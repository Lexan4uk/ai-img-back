# 02. Модульная архитектура

## Принципы

- **Clean Architecture:** разделение на слои (Data → Domain → Presentation)
- **Dependency Inversion:** все зависимости направлены к центру (Domain)
- **Single Responsibility:** один модуль — одна ответственность
- **Подготовка к расширению:** абстракции для смены БД (SQLite → PostgreSQL)

## Описание модулей

### :app-desktop
Точка входа десктопного приложения. Инициализация окна Compose, DI-контейнера,
роутера навигации и менеджера тем.

### :feature-* (9 модулей)
Каждый feature-модуль реализует один экран:

| Модуль | Экран | Содержимое |
|--------|-------|-----------|
| `:feature-dashboard` | S-01 | DashboardScreen, DashboardViewModel |
| `:feature-filters` | S-02 | FiltersScreen, FiltersViewModel |
| `:feature-devices` | S-03 | DevicesScreen, DevicesViewModel |
| `:feature-instances` | S-04 | InstancesScreen, InstancesViewModel |
| `:feature-users` | S-05 | UsersScreen, UsersViewModel |
| `:feature-history` | S-06 | HistoryScreen, HistoryViewModel |
| `:feature-favorites` | S-07 | FavoritesScreen, FavoritesViewModel |
| `:feature-session-view` | S-08 | SessionViewScreen, SessionViewViewModel |
| `:feature-visualization` | S-09 | VisualizationDialog, VisualizationViewModel |

Каждый feature-модуль содержит:
- `Screen.kt` — Composable-функция экрана
- `ViewModel.kt` — управление состоянием (StateFlow)
- `State.kt` — sealed class / data class состояния экрана
- `Components.kt` — локальные UI-компоненты (если нужны)

### :domain
Чистый Kotlin-модуль без зависимостей от фреймворков.

**Entities:** Доменные модели (data class):
```
User, DeviceModel, DeviceInstance, Session, Swipe, SwipePoint, Favorite
```

**Use Cases:** Бизнес-логика в виде интеракторов:
```
GetSessionsUseCase, FilterSessionsUseCase, GetSwipesUseCase,
ToggleFavoriteUseCase, GetStatisticsUseCase, SearchEntitiesUseCase
```

**Repository Interfaces:** Абстракции для слоя данных:
```
interface SessionRepository {
    suspend fun getAll(page: Int, pageSize: Int): List<Session>
    suspend fun getById(id: String): Session?
    suspend fun filter(criteria: FilterCriteria): List<Session>
    suspend fun getRecent(count: Int): List<Session>
}
```

### :data
Реализация репозиториев. Разделён на подмодули:

- **:data-sqlite** — реализация через SQLDelight (by default)
- **:data-postgres** — реализация через JDBC/Exposed (v2+, зарезервировано)
- **:data-remote** — Ktor-клиент для REST API сервера
- **:data-sync** — логика инкрементальной синхронизации

### :core
Утилитарный модуль с общими зависимостями:
- Конфигурация Koin (DI modules)
- Утилиты форматирования
- Ресурсы и строки

## Граф зависимостей

```
:app-desktop
  ├── :feature-* (все feature модули)
  ├── :domain
  ├── :data
  │   ├── :data-sqlite
  │   ├── :data-postgres  (v2+)
  │   ├── :data-remote
  │   └── :data-sync
  └── :core

:feature-*
  ├── :domain
  └── :core

:data-*
  ├── :domain
  └── :core

:domain
  └── (нет зависимостей — чистый Kotlin)

:core
  └── (нет зависимостей — чистый Kotlin)
```

> **Правило:** `:feature-*` модули **не зависят** от `:data-*`.
> Вся работа с данными идёт через интерфейсы из `:domain`, реализации инжектируются через Koin.
