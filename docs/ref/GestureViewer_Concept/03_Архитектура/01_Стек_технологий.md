# 01. Технологический стек

## Платформы

| Платформа | Архитектура | Формат дистрибуции |
|-----------|-------------|-------------------|
| Ubuntu (Linux) | x86_64 | .deb / AppImage |
| Windows | x86_64 | .msi / .exe installer |

## Стек

| Слой | Технология | Версия | Назначение |
|------|-----------|--------|------------|
| Язык | Kotlin | 1.9.x+ | Основной язык разработки |
| UI Framework | Compose Multiplatform (Desktop) | 1.6.x+ | Декларативный UI |
| Сборка | Gradle (Kotlin DSL) | 8.x | Система сборки |
| Локальная БД | SQLite | 3.x | Хранение данных (v1) |
| ORM / SQL | SQLDelight | 2.x | Типобезопасные SQL-запросы |
| DI | Koin | 3.x | Dependency Injection |
| Навигация | Voyager / Decompose | — | Навигация между экранами |
| Сеть | Ktor Client | 2.x | HTTP-клиент для синхронизации |
| Сериализация | kotlinx.serialization | 1.6.x | JSON сериализация |
| Логирование | SLF4J + Logback | — | Логи приложения |
| Тестирование | JUnit 5 / Kotest | — | Unit и Integration тесты (UI тесты не требуются) |

> **Примечание по тестированию:**
> - UI-тесты не требуются.
> - Интеграционные тесты обязательны.
> - Выбор фреймворка (JUnit или Kotest) остаётся на усмотрение разработчика.

## Обоснование выбора

### Kotlin Multiplatform + Compose Desktop
- Единая кодовая база для Linux и Windows
- Декларативный UI с реактивным state management
- Нативная производительность (JVM → native packaging)
- Сильная экосистема (корутины, Flow, сериализация)

### SQLite через SQLDelight
- Zero-configuration: не требует установки дополнительного ПО
- SQLDelight генерирует типобезопасные Kotlin-классы из SQL
- Хорошо подходит для read-heavy сценария (визуализация)
- Абстракция позволит добавить PostgreSQL в будущем

### Ktor Client
- Kotlin-native HTTP клиент
- Поддержка корутин из коробки
- Лёгкий, без лишних зависимостей
- Подходит для REST API и может быть расширен до gRPC

## Нефункциональные требования

| Требование | Метрика |
|-----------|---------|
| Открытие списка сессий (≤1000) | ≤ 1 сек |
| Отрисовка визуализации свайпа (≤500 точек) | ≤ 0.5 сек |
| Применение фильтров | Реальное время (debounce 300мс) |
| Анимация воспроизведения свайпа | 60 FPS |
| Запуск приложения | ≤ 3 сек |
| Потребление RAM | ≤ 512 МБ (при 100К сессий) |

## Поддержка тем

- Светлая и тёмная тема
- Переключение через UI (header)
- Выбор сохраняется в локальные настройки (Preferences)
- Material 3 Design System
