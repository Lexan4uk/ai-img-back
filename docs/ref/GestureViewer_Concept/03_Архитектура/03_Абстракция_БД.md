# 03. Абстракция базы данных и стратегия расширения

## Текущее состояние (v1): SQLite

В первой версии приложение работает исключительно с локальной SQLite.
Файл БД располагается в директории приложения:

```
~/.gestureviewer/data.db          (Linux)
%APPDATA%\GestureViewer\data.db   (Windows)
```

**Реализация:** SQLDelight генерирует Kotlin-классы из `.sq` файлов.

---

## Будущее расширение (v2+): PostgreSQL

### Сценарий

В дальнейшем добавляется возможность подключения к локальному PostgreSQL-серверу.
Все данные синхронизируются с центрального сервера в PostgreSQL.
Пользователь выбирает backend в настройках.

### Архитектурная подготовка

### Реализация через Koin

```kotlin
// При запуске приложения:
val dbModule = when (settings.databaseBackend) {
    Backend.SQLITE -> sqliteModule      // binds SqliteSessionRepo, ...
    Backend.POSTGRES -> postgresModule  // binds PgSessionRepo, ...
}

startKoin {
    modules(coreModule, domainModule, dbModule, featureModules)
}
```

### Синхронизация (v2+)

```
┌────────────────┐      REST / gRPC       ┌─────────────┐
│  Центральный   │◄────────────────────── │  Desktop    │
│  сервер        │ ─────────────────────► │  клиент     │
│                │  (данные свайпов)       │             │
└────────────────┘                         └──────┬──────┘
                                                  │
                                           ┌──────▼──────┐
                                           │ PostgreSQL  │
                                           │ (локальный) │
                                           └─────────────┘
```

**Режимы синхронизации:**
1. **Полная синхронизация** — загрузка всей БД (первый запуск)
2. **Инкрементальная** — только новые сессии (по `created_at > last_sync`)
3. **Ручной запуск** — кнопка «Синхронизировать» в UI

**Статус синхронизации** отображается в Footer:
```
PostgreSQL ● Подключено │ Синхр: 12 450 / 12 450 (100%) │ 12.02.2026 14:32
```

## Миграции

### SQLite → PostgreSQL

При первом переключении на PostgreSQL:
1. Создание схемы в PostgreSQL (DDL)
2. Миграция данных из SQLite в PostgreSQL (одноразовая)
3. Переключение приложения на новый backend

### Схема данных

Идентична для обоих бэкендов. Различия только в типах данных:

| Поле | SQLite | PostgreSQL |
|------|--------|-----------|
| ID | TEXT | UUID |
| Дата | TEXT (ISO-8601) | TIMESTAMPTZ |
| JSON (tags) | TEXT | JSONB |
| Числа | REAL | DOUBLE PRECISION |
| Автоинкремент | INTEGER PRIMARY KEY | SERIAL / BIGSERIAL |
