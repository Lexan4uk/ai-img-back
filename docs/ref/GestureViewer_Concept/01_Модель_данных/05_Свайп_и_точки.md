# 05. Сущности: Свайп (Swipe) и Точка свайпа (SwipePoint)

## Назначение

**Swipe** — одно непрерывное касание пальца (от touch-down до touch-up).
**SwipePoint** — отдельная точка трека в рамках свайпа, фиксируемая с частотой сенсора.

Эти две сущности содержат основные данные для визуализации и анализа.

---

## Swipe — Свайп

### Поля

| Поле | Тип | Обязательное | Описание |
|------|-----|:---:|----------|
| `id` | TEXT (UUID) | ✓ | Уникальный идентификатор свайпа |
| `session_id` | TEXT (FK) | ✓ | Ссылка на Session |
| `pressure_quality` | TEXT (ENUM) | ✓ | Оценка давления: `valid`, `binary`, `invalid` |
| `radius_quality` | TEXT (ENUM) | ✓ | Оценка радиусов: `valid`, `circular`, `point`, `invalid` |
| `order_index` | INTEGER | ✓ | Порядковый номер свайпа в сессии (0-based) |

### Двухосевая оценка качества

Оценки присваиваются сервером на этапе постобработки. Каждый свайп
оценивается **независимо** по двум параметрам: давление и радиусы.

> Подробное описание, примеры данных и правила определения —
> см. [07_Оценки_свайпов.md](07_Оценки_свайпов.md).

```
┌────────────────────────────────────────────────────────────────────┐
│             Оценка давления (PressureQuality)                     │
├──────────┬───────────────────────────────────────┬────────────────┤
│ Значение │ Условие                               │ Бейдж          │
├──────────┼───────────────────────────────────────┼────────────────┤
│ valid    │ Реальные значения 0.0–1.0             │ 🟢 Зачётный    │
│ binary   │ Только 0 или 1                        │ 🟡 Двоичный    │
│ invalid  │ Неконсистентные данные                 │ 🔴 Незачётный  │
└──────────┴───────────────────────────────────────┴────────────────┘

┌────────────────────────────────────────────────────────────────────┐
│             Оценка радиусов (RadiusQuality)                       │
├──────────┬───────────────────────────────────────┬────────────────┤
│ Значение │ Условие                               │ Бейдж          │
├──────────┼───────────────────────────────────────┼────────────────┤
│ valid    │ r1 ≠ r2, оба > 0                     │ 🟢 Зачётный    │
│ circular │ r1 = r2, оба > 0                     │ 🔵 Круглый     │
│ point    │ Радиусы отсутствуют или = 1           │ 🟠 Точечный    │
│ invalid  │ Неконсистентные данные                 │ 🔴 Незачётный  │
└──────────┴───────────────────────────────────────┴────────────────┘
```

### Итоговый статус свайпа

Свайп считается **зачётным**, если **обе** оценки ≠ `invalid`.
Свайп считается **незачётным**, если хотя бы одна оценка = `invalid`.

### Бизнес-правила

- `order_index` определяет порядок отображения (слева направо на S-08)
- Оценки давления и радиусов используются для фильтрации на S-02 (мультивыбор)
- Оценки определяют цветовую индикацию бейджей на карточке свайпа (S-08)

## SwipePoint — Точка свайпа

### Поля

| Поле | Тип | Обязательное | Описание |
|------|-----|:---:|----------|
| `id` | TEXT (UUID) | ✓ | Уникальный идентификатор точки |
| `swipe_id` | TEXT (FK) | ✓ | Ссылка на Swipe |
| `x` | REAL | ✓ | Координата X на экране устройства (пиксели) |
| `y` | REAL | ✓ | Координата Y на экране устройства (пиксели) |
| `r1` | REAL | ✗ | Радиус 1 эллипса касания (полуось) |
| `r2` | REAL | ✗ | Радиус 2 эллипса касания (полуось) |
| `pressure` | REAL | ✗ | Давление пальца: 0.0 – 1.0 |
| `timestamp` | INTEGER (BIGINT) | ✓ | Unix-время в миллисекундах |

### Диапазоны значений

| Параметр | Мин | Макс | Примечание |
|----------|-----|------|-----------|
| `x`, `y` | 0 | Разрешение экрана | Зависит от устройства |
| `r1`, `r2` | 0.0 | ~50.0 | В пикселях; 0 = данные отсутствуют |
| `pressure` | 0.0 | 1.0 | 0 = не поддерживается датчиком |
| `timestamp` | — | — | Монотонно возрастающий в рамках свайпа |

### Визуализация точки (экран S-09)

```
              Обводка: всегда непрозрачная
             ┌───────────────────────┐
             │ ╭─────────────────╮   │
             │ │                 │   │
  r2 (ось Y) │ │    ● (x, y)     │   │  ← Заливка зависит от pressure
             │ │                 │   │
             │ ╰─────────────────╯   │
             └───────────────────────┘
                   r1 (ось X)
```

### Цветовая кодировка давления (pressure → цвет заливки)

```
pressure:  0.0           0.25          0.50          0.75          1.0
           │              │              │              │             │
Цвет:    Синий ─────── Синий ─────── Переход ────── Оранж-кр. ── Красный
Прозрач.: 100%           75%            50%            25%           0%
           │              │              │              │             │
           ▼              ▼              ▼              ▼             ▼
        (только       (слабая        (средняя       (сильная    (непрозр.
         обводка)     заливка)        заливка)       заливка)    заливка)
```

> **Важно:** При `pressure = 0` заливка полностью прозрачна,
> но контур (обводка) эллипса всегда отображается.

## Связи

```
Session ──┐
          │ 1 : N (по order_index)
          ▼
        Swipe ──┐
                │ 1 : N (по timestamp)
                ▼
            SwipePoint
```

## Отображение в UI

| Экран | Swipe | SwipePoint |
|-------|-------|------------|
| S-08 (Просмотр сессии) | Карточки свайпов с порядковым номером, двумя бейджами оценок (давление + радиус), кол-вом точек и сырыми данными | Отображение x, y, r1, r2, pressure, timestamp в карточке |
| S-09 (Визуализация) | Инфо-панель: номер, оценки давления и радиуса, кол-во точек, длительность | Графическое отображение: точки, эллипсы, заливка давления |
| S-02 (Фильтры) | Фильтры по `pressure_quality` и `radius_quality` (мультивыбор) | Фильтр по наличию вариативности pressure/radius |
