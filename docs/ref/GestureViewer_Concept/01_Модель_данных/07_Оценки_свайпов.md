# 07. Оценки свайпов (Swipe Quality)

## Назначение

Описание двухосевой системы оценки качества данных свайпа. Оценки
присваиваются сервером на этапе постобработки после получения сырых данных
с устройств. Каждый свайп оценивается **независимо** по двум параметрам:
**давление** и **радиусы**.

---

## Ось 1: Качество давления (PressureQuality)

Оценка консистентности значений `pressure` по всем точкам свайпа.

### Перечисление

| Значение | Название | Описание |
|----------|----------|----------|
| `valid` | Зачётный | Реальные вариативные значения 0.0–1.0 (например 0.59, 0.63, 0.57) |
| `binary` | Двоичный | Все значения только 0 или 1 (датчик не передаёт градации) |
| `invalid` | Незачётный | Неконсистентные данные (часть точек имеет реальные значения, часть — нули или пропуски) |

### Примеры данных

```
Зачётный (valid):
  pressure: 0.59  0.58  0.57  0.56  0.57
  → Все значения в диапазоне, плавное изменение

Двоичный (binary):
  pressure: 1  1  1  1  1  1
  → Датчик отдаёт только «касание/нет»

Незачётный (invalid):
  pressure: 0.59  0.58  0  0  0.57
  → Часть точек потеряла данные, неконсистентность
```

### Правила определения

```
Если хотя бы одна точка имеет pressure, а другая — нет (или 0)
  И при этом остальные значения вариативны (не все 0 и не все 1):
  → invalid

Если все значения pressure ∈ {0, 1} (только бинарные):
  → binary

Если значения вариативны и присутствуют у всех точек:
  → valid
```

## Ось 2: Качество радиусов (RadiusQuality)

Оценка консистентности значений `r1` и `r2` по всем точкам свайпа.

### Перечисление

| Значение | Название | Описание |
|----------|----------|----------|
| `valid` | Зачётный | r1 ≠ r2, оба > 0 — полноценный эллипс касания |
| `circular` | Круглый | r1 = r2 у всех точек (типично для Xiaomi) — круговое касание |
| `point` | Точечный | Радиусы отсутствуют или равны 1 — касание фиксируется как точка |
| `invalid` | Незачётный | Неконсистентные данные (часть точек имеет радиусы, часть — нет) |

### Примеры данных

```
Зачётный (valid):
  R1:  7   8   9  10   9  10  11  12
  R2:  6   6   7   7   9   8   9  10
  → R1 ≠ R2, оба > 0, полноценный эллипс

Круглый (circular):
  R1:  7   8   9  10   9  10  11  12
  R2:  7   8   9  10   9  10  11  12
  → R1 = R2 во всех точках

Точечный (point):
  R1:  1   1   1   1   1   1   1
  R2:  1   1   1   1   1   1   1
  → Радиусы номинальные, касание = точка

Незачётный (invalid):
  R1:  7   8   9  10   9  10  11  12
  R2:  6   6   0   0   0   0   9  10
  → Часть значений R2 потеряна
```

### Правила определения

```
Если у части точек r2 отсутствует или = 0, а у части — валидные значения:
  → invalid

Если r1 = r2 у всех точек (и оба > 1):
  → circular

Если все радиусы = 1 или отсутствуют:
  → point

Если r1 ≠ r2, оба > 0 у всех точек:
  → valid
```

## Итоговый статус свайпа

Свайп считается **зачётным**, если **обе** оценки ≠ `invalid`:

```
┌────────────────────────┬──────────────────────────────────┐
│   PressureQuality      │         RadiusQuality            │
│                        │  valid  circular  point  invalid │
├────────────────────────┼────────┬─────────┬──────┬────────┤
│ valid                  │   ✅   │   ✅    │  ✅  │   ❌   │
│ binary                 │   ✅   │   ✅    │  ✅  │   ❌   │
│ invalid                │   ❌   │   ❌    │  ❌  │   ❌   │
└────────────────────────┴────────┴─────────┴──────┴────────┘

 ✅ = зачётный свайп (учитывается в консистентности)
 ❌ = незачётный свайп (хотя бы одна ось = invalid)
```

---

## Консистентность сессии

Агрегированный показатель качества данных на уровне сессии:

```
consistency = (кол-во зачётных свайпов) / (общее кол-во свайпов) × 100%
```

Свайп зачётный, если `pressure_quality ≠ invalid` **И** `radius_quality ≠ invalid`.

| Значение | Визуальная индикация |
|----------|---------------------|
| 100% | ✅ Зелёный — Все данные корректны |
| 50–99% | ⚠️ Жёлтый — Есть проблемные свайпы |
| < 50% | 🔴 Красный — Большинство свайпов некорректны |

## Цветовая кодировка оценок

### Оценка давления (PressureQuality)

| Значение | Бейдж | Цвет |
|----------|-------|------|
| `valid` | 🟢 Зачётный | Зелёный |
| `binary` | 🟡 Двоичный | Жёлтый |
| `invalid` | 🔴 Незачётный | Красный |

### Оценка радиусов (RadiusQuality)

| Значение | Бейдж | Цвет |
|----------|-------|------|
| `valid` | 🟢 Зачётный | Зелёный |
| `circular` | 🔵 Круглый | Синий |

| `invalid` | 🔴 Незачётный | Красный |

## Источник данных

Оценки вычисляются **на сервере** при постобработке и приходят вместе с данными свайпа
через API. Desktop-клиент отображает готовые значения без повторного вычисления.
