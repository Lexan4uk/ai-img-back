# 02. Сущность: Пользователь (User)

## Назначение

Представляет человека, участвующего в сборе данных свайпов.
Каждый пользователь идентифицируется уникальным ID и имеет демографические атрибуты.

## Поля

| Поле | Тип | Обязательное | Описание |
|------|-----|:---:|----------|
| `id` | TEXT (UUID) | ✓ | Уникальный идентификатор пользователя, приходит с сервера |
| `name` | TEXT | ✓ | Имя / псевдоним пользователя |
| `gender` | TEXT | ✗ | Пол: `"male"`, `"female"`, `"other"` |
| `age` | INTEGER | ✗ | Возраст на момент регистрации |
| `activity_type` | TEXT | ✗ | Вид деятельности (врач, программист, тренер и т.д.) |
| `created_at` | TEXT (ISO-8601) | ✓ | Дата создания записи |

## Бизнес-правила

- Имя используется в UI как основной идентификатор (отображается в списках, карточках сессий)
- Пол и возраст используются для фильтрации на экране S-02 (Расширенные фильтры)
- Возрастной диапазон фильтрации: слайдер от–до

## Связи

```
User ──┐
       │ 1 : N
       ▼
    Session
```

- Один пользователь может иметь 0..N сессий
- Удаление пользователя из избранного не удаляет его сессии

## Отображение в UI

| Экран | Что отображается |
|-------|-----------------|
| S-05 (По пользователям) | Таблица: ID, имя, пол, возраст, кол-во сессий, кол-во устройств, дата последней сессии |
| S-06 (История сессий) | Имя пользователя в карточке сессии |
| S-08 (Просмотр сессии) | Имя пользователя в шапке сессии |
| S-07 (Избранное) | Вкладка «Избранные пользователи» |
| S-02 (Фильтры) | Поиск с автодополнением по имени/ID |

## Агрегаты для UI

| Агрегат | Описание | Где используется |
|---------|----------|-----------------|
| `session_count` | Общее количество сессий пользователя | S-05, S-01 |
| `device_count` | Кол-во уникальных экземпляров устройств | S-05 |
| `last_session_date` | Дата последней сессии | S-05 |
