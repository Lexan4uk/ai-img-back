# 04. Сущность: Сессия (Session)

## Назначение

Центральная сущность системы. Сессия — это единичный сеанс записи свайпов
конкретного пользователя на конкретном устройстве. Содержит контекст записи
(рука, палец, состояние, теги) и упорядоченный набор свайпов.

## Поля

| Поле | Тип | Обязательное | Описание |
|------|-----|:---:|----------|
| `id` | TEXT (UUID) | ✓ | Уникальный идентификатор сессии |
| `user_id` | TEXT (FK) | ✓ | Ссылка на User |
| `device_instance_id` | TEXT (FK) | ✓ | Ссылка на DeviceInstance |
| `hand` | TEXT | ✓ | Рука: `"left"` / `"right"` |
| `finger` | TEXT | ✓ | Палец: `"index"`, `"middle"`, `"ring"`, `"pinky"`, `"thumb"` |
| `state` | TEXT | ✗ | Состояние пользователя (произвольный текст) |
| `tags` | TEXT (JSON array) | ✗ | Массив произвольных тегов: `["тег1", "тег2"]` |
| `created_at` | TEXT (ISO-8601) | ✓ | Дата и время проведения сессии |

## Перечисления

### Hand (Рука)
| Значение | Отображение |
|----------|-------------|
| `left` | Левая |
| `right` | Правая |

### Finger (Палец)
| Значение | Отображение |
|----------|-------------|
| `thumb` | Большой |
| `index` | Указательный |
| `middle` | Средний |
| `ring` | Безымянный |
| `pinky` | Мизинец |

## Связи

```
User ──► Session ◄── DeviceInstance
              │
              │ 1 : N
              ▼
            Swipe
```

## Индикатор консистентности

На уровне сессии вычисляется агрегированный показатель качества данных.
Свайп считается **зачётным**, если **обе** оценки ≠ `invalid`
(подробнее — см. [07_Оценки_свайпов.md](07_Оценки_свайпов.md)):

```
зачётный_свайп = (pressure_quality ≠ invalid) И (radius_quality ≠ invalid)
consistency    = (кол-во зачётных свайпов) / (общее кол-во свайпов) × 100%
```

| Значение | Визуальная индикация |
|----------|---------------------|
| 100% | ✅ Зелёный — Все данные корректны |
| 50–99% | ⚠️ Жёлтый — Есть проблемные свайпы |
| < 50% | 🔴 Красный — Большинство свайпов некорректны |

## Отображение в UI

| Экран | Контекст |
|-------|----------|
| S-01 (Dashboard) | Блок «Последние 5–10 сессий», карточки статистики |
| S-02 (Фильтры) | Результаты фильтрации — таблица сессий |
| S-03 (По устройствам) | Переход к сессиям модели |
| S-04 (По экземплярам) | Переход к сессиям экземпляра |
| S-05 (По пользователям) | Переход к сессиям пользователя |
| S-06 (История) | Полный хронологический список |
| S-07 (Избранное) | Вкладка «Избранные сессии» |
| S-08 (Просмотр сессии) | Главный экран — детальный просмотр |

## Агрегаты для UI

| Агрегат | Описание |
|---------|----------|
| `swipe_count` | Количество свайпов в сессии |
| `consistency_percent` | Процент зачётных свайпов (обе оценки ≠ `invalid`) |
| `pressure_valid_count` | Кол-во свайпов с `pressure_quality = valid` |
| `pressure_binary_count` | Кол-во свайпов с `pressure_quality = binary` |
| `pressure_invalid_count` | Кол-во свайпов с `pressure_quality = invalid` |
| `radius_valid_count` | Кол-во свайпов с `radius_quality = valid` |
| `radius_circular_count` | Кол-во свайпов с `radius_quality = circular` |
| `radius_point_count` | Кол-во свайпов с `radius_quality = point` |
| `radius_invalid_count` | Кол-во свайпов с `radius_quality = invalid` |
| `device_model_name` | Название модели (через DeviceInstance → DeviceModel) |
| `device_brand` | Бренд (через DeviceInstance → DeviceModel) |
| `user_name` | Имя пользователя (через User) |
