# 05. Интеграция с AI-провайдерами

## Общая картина

Система генерирует изображения по текстовому описанию пользователя.
Подключены два AI-провайдера — **OpenAI** (платный, высокое качество) и **Stability AI** (платный, доступен без VPN из РФ). Картинка сохраняется на диск сервера и регистрируется в базе данных.

```
                                         ┌───────────────────────┐
                                    ┌───►│  OpenAI                │
                                    │    │  gpt-image-1-mini      │
┌────────┐   ┌──────────┐   ┌──────┴──┐ │  gpt-image-1.5         │
│ Клиент │──►│ Генера-  │──►│ Provider │ └───────────────────────┘
│        │   │   ция    │   │  Router  │
└────────┘   └──────────┘   └──────┬──┘ ┌───────────────────────┐
                                    │    │  Stability AI          │
                                    └───►│  Stable Image Core     │
                                         │  (из РФ без VPN)       │
                                         └───────────────────────┘
```

> **Язык промптов:** Stability AI **не понимает русский** — промпты должны быть на английском. OpenAI понимает русский, но качество выше на английском. **Рекомендация: пользователям писать промпты на английском языке** — это гарантирует лучший результат на обоих провайдерах.

## 1. Подключённые провайдеры

### 1.1. OpenAI GPT Image (основной)

| Параметр         | Значение                                             |
| ---------------- | ---------------------------------------------------- |
| Модели           | `gpt-image-1-mini`, `gpt-image-1.5`                  |
| Качество         | Высокое (лучшее среди подключённых)                  |
| Стоимость        | ~$0.02–0.19 / картинка (зависит от качества/размера) |
| Скорость         | ~8–30 секунд (зависит от модели)                     |
| Доступ из России | Нет — нужен **прокси** или VPN                       |
| Язык промптов    | Русский и английский (английский — лучше качество)   |

> **Выбор модели:** клиент может указать конкретную модель OpenAI в запросе (поле `model`). Модель `gpt-image-1-mini` — быстрее и дешевле, `gpt-image-1.5` — выше качество.

**Поддерживаемые размеры (aspect ratio):**

| Aspect ratio | Размер в пикселях | Ориентация |
| ------------ | ----------------- | ---------- |
| `1:1`        | 1024 × 1024       | Квадрат    |
| `1:1.5`      | 1024 × 1536       | Портрет    |
| `1.5:1`      | 1536 × 1024       | Альбом     |
| `auto`       | 1024 × 1024       |            |

> Всего **3 фиксированных размера** + авто. Если клиент передаёт другое соотношение — система автоматически выставит `auto`.

### 1.3. Сравнение провайдеров

| Критерий           | OpenAI GPT Image            | Stability AI                      |
| ------------------ | --------------------------- | --------------------------------- |
| Качество           | Отличное                    | Хорошее                           |
| Скорость           | ~8–30 сек (зависит от модели) | ~5–10 сек                       |
| Стоимость          | ~$0.02–0.19 / картинка      | ~$0.03 / картинка (3 кредита)     |
| Варианты размеров  | 3 + auto                    | 9 вариантов                       |
| Доступ из России   | Нет (нужен прокси)          | Да (работает напрямую)            |

- Если нужно максимальное качество и прокси настроен — используйте **OpenAI**
- В режиме `auto` система сама выберет доступного провайдера

## 2. Размеры изображений (aspect ratio)

Пользователь выбирает **соотношение сторон** (aspect ratio), а не пиксели.
Система сама конвертирует ratio в формат, который понимает конкретный провайдер.

**Пример:** клиент отправляет `"aspectRatio": "1:1.5"`:

- OpenAI получит `size: "1024x1536"` (портрет)
- Stability получит `aspect_ratio: "2:3"` (ближайшее допустимое)

Если aspect ratio не указан — используется **1:1** (квадрат) по умолчанию.

## 3. Настраиваемый шаблон промпта

Промпт, отправляемый в AI, формируется по шаблону. Шаблон настраивается в одном месте — `application.properties`:

```properties
ai.prompt-template=Create an image of type: {type}. Style: {style}. Description: {prompt}. Avoid unnecessary text in the image.
```

**Переменные:**
| Переменная | Что подставляется | Пример |
| ---------- | ----------------- | ------ |
| `{type}` | Промпт типа изображения (из БД) | «Фотография», «Иллюстрация» |
| `{style}` | Промпт стиля (из БД) | «Минимализм», «Айвазовский» |
| `{prompt}` | Текст пользователя | «Кузнец куёт меч у моря» |

Шаблон **единый** для всех провайдеров — одинаковый промпт идёт и в OpenAI, и в Stability AI.

## 4. Выбор модели на клиенте

Клиент предоставляет пользователю **dropdown выбор модели**. Каждая модель определяет:

- Провайдер (`provider`) и имя модели (`model`) для бэкенда
- Допустимые **aspect ratio** (dropdown обновляется при смене модели)
- Нужен ли прокси (`routingMode: DIRECT / PROXY`)

### Доступные модели в клиенте

| Модель в UI                  | provider    | model               | Aspect ratios                               |
| ---------------------------- | ----------- | -------------------- | ------------------------------------------- |
| OpenAI GPT Image 1 Mini     | `openai`    | `gpt-image-1-mini`  | 1:1, 1:1.5, 1.5:1, auto                    |
| OpenAI GPT Image 1.5        | `openai`    | `gpt-image-1.5`     | 1:1, 1:1.5, 1.5:1, auto                    |
| Stability Stable Image Core | `stability` | `stable-image-core`  | 1:1, 16:9, 9:16, 3:2, 2:3, 4:5, 5:4, 21:9, 9:21 |

При смене модели клиент автоматически сбрасывает aspect ratio на значение по умолчанию, если текущий ratio не поддерживается новой моделью.

Реестр моделей хранится в `AiModelRegistry` на клиенте. Для добавления новой модели — достаточно добавить запись в реестр.

## 5. Дедупликация запросов

### Зачем

Каждая генерация стоит денег. Если один и тот же запрос отправить дважды — система обнаружит дубликат и **не будет платить повторно**.

### Как это работает

При создании запроса система вычисляет **SHA-256 хеш** из итогового промпта и параметров генерации:

```
hash = SHA-256( "Create an image of type: Photo. Style: Minimalism. ..." + "|" + '{"aspectRatio":"1:1"}' )
      → "a3f8c1d2e5b7..."
```

Этот хеш — уникальный отпечаток запроса. Если промпт, тип, стиль или aspect ratio отличаются хотя бы немного — хеш будет другим, и запрос считается новым.

### Что влияет на хеш

| Изменение | Хеш изменится? | Будет новая генерация? |
| --------- | -------------- | --------------------- |
| Другой текст промпта | Да | Да |
| Другой тип изображения | Да | Да |
| Другой стиль | Да | Да |
| Другой aspect ratio | Да | Да |
| Другой пользователь, всё остальное то же | Нет | Нет (SKIP) |
| Всё одинаковое, повторный запрос | Нет | Нет (SKIP) |

### Режимы

- **SKIP** (по умолчанию) — дубликаты пропускаются бесплатно. Ранее созданная картинка уже есть в базе.
- **OVERWRITE** — генерация выполняется даже для дубликатов. Полезно, когда хочется получить другой вариант по тому же описанию.

### Пример экономии

Пользователь генерирует 3 типа x 2 стиля = **6 картинок**. Из них 2 уже были сгенерированы ранее.

| Режим | Платных генераций | Пропущено | Итого расход |
| ----- | ---------------- | --------- | ------------ |
| SKIP | 4 | 2 бесплатно | ~$0.08–0.76 |
| OVERWRITE | 6 | 0 | ~$0.12–1.14 |

> Подробнее о хеше и дедупликации — см. [07_Генерация.md](../01_Модель_данных/07_Генерация.md).

## 6. Архитектура провайдеров в коде

### 6.1. Паттерн Strategy

```
┌──────────────────────────────┐
│   ImageAiProvider             │  (интерфейс)
│   + generate(prompt,          │
│     aspectRatio, model)       │
│     → byte[]                  │
│   + providerName() → String   │
│   + isAvailable() → boolean   │
└─────────┬────────────────────┘
          │
   ┌──────┼────────────┐
   ▼                    ▼
┌────────────────┐ ┌──────────────────┐
│ OpenAiProvider  │ │StabilityProvider  │
│                 │ │                   │
│ JSON API        │ │ multipart/form-data│
│ Bearer token    │ │ Bearer token       │
│ base64 ответ    │ │ байты напрямую     │
└────────────────┘ └──────────────────┘
```

Метод `generate` принимает 3 параметра: промпт, aspect ratio и **имя модели** (nullable — если null, используется модель по умолчанию для провайдера).

Добавление нового провайдера = один новый Java-класс, реализующий интерфейс `ImageAiProvider`. Никакие другие файлы менять не нужно — Spring автоматически подхватит новый провайдер.

### 6.2. Логика выбора провайдера

```
Пользователь указал provider?
├── Да → Использовать указанного
│        ├── Доступен? → Генерировать
│        └── Недоступен? → Ошибка
│
└── Нет (provider = null / "auto") → Автоматический выбор
    ├── Провайдер по умолчанию доступен? → Использовать его
    └── Иначе → Любой доступный
```

Когда пользователь явно указал провайдер — система **не переключается** на другой автоматически. Это важно для предсказуемости: пользователь контролирует расходы.

## 7. Прокси-сервер

### 7.1. Зачем нужен прокси

OpenAI **заблокирован** в России. Если сервер находится в РФ, запросы к OpenAI не пройдут без прокси. Stability AI **работает напрямую** — прокси для него не нужен.

```
Без прокси:
  Сервер (Россия) ──► OpenAI API     → Заблокировано ✘
  Сервер (Россия) ──► Stability API  → Работает ✔

С прокси:
  Сервер ──► Прокси (IP в США) ──► OpenAI API → Работает ✔
```

### 7.2. Настройка

```properties
ai.proxy.enabled=true
ai.proxy.host=proxy-server.proxy-seller.io
ai.proxy.port=10000
ai.proxy.username=user123
ai.proxy.password=pass456
ai.proxy.type=HTTP
```

Рекомендуются **datacenter-прокси** от proxy-seller.io — быстрые, дешёвые (~$1/IP/мес), AI-провайдеры их не блокируют.

## 8. Конфигурация (application.properties)

Все настройки AI собраны в одном файле. Здесь же хранятся API-ключи (через переменные окружения или напрямую для разработки):

```properties
# --- Шаблон промпта ---
ai.prompt-template=Create an image of type: {type}. Style: {style}. Description: {prompt}. Avoid unnecessary text.

# --- OpenAI GPT Image ---
ai.openai.api-key=${OPENAI_API_KEY:}
ai.openai.model=gpt-image-1.5
ai.openai.timeout-seconds=90
ai.openai.max-retries=3

# --- Stability AI ---
ai.stability.api-key=${STABILITY_API_KEY:}
ai.stability.timeout-seconds=90
ai.stability.max-retries=2
ai.stability.output-format=png

# --- Прокси ---
ai.proxy.enabled=false
ai.proxy.host=
ai.proxy.port=10000

# --- Хранение ---
ai.images.output-dir=generated-imgs

# --- Провайдер по умолчанию ---
ai.default-provider=openai
# Варианты: openai, stability, auto
```

### Переменные окружения

| Переменная          | Назначение        | Где получить                       |
| ------------------- | ----------------- | ---------------------------------- |
| `OPENAI_API_KEY`    | Ключ OpenAI       | platform.openai.com/api-keys       |
| `STABILITY_API_KEY` | Ключ Stability AI | platform.stability.ai/account/keys |

## 9. Хранение изображений

Сгенерированные картинки сохраняются на диск сервера:

```
ai-img-back/
├── generated-imgs/           ← папка с картинками
│   ├── a1b2c3d4-...-e5f6.png
│   ├── g7h8i9j0-...-k1l2.webp
│   └── ...
```

- Имя файла = случайный UUID + расширение (`.png`, `.jpg`, `.webp`)
- Поддерживаемые форматы: **PNG, JPEG, WebP**
- В базе хранится относительный путь: `generated-imgs/a1b2c3d4.png`
- Папка создаётся автоматически при запуске сервера

**Примерный расход диска:** одна картинка ≈ 200–500 КБ. 10 000 картинок ≈ 2–5 ГБ.

> **⚠️ Узкое место:** хранилище растёт бесконтрольно. Нет механизма очистки старых файлов,
> нет лимита на общий размер, нет уведомления при заполнении диска.
> При активном использовании (100 картинок/день) диск заполнится за несколько месяцев.
> Рекомендуется: TTL-политика, миграция в S3/MinIO, мониторинг свободного места.

## 10. Обработка ошибок

### 10.1. Основные ситуации

| Ситуация                          | Что делает система                                                                                                        |
| --------------------------------- | ------------------------------------------------------------------------------------------------------------------------- |
| **Отключение электричества**      | При рестарте незавершённые запросы автоматически помечаются как ошибка. Данные в БД сохранены. Можно повторить генерацию. |
| **Потеря интернета**              | 3 повторные попытки (через 2 → 4 → 8 сек). Если не помогло — ошибка.                                                      |
| **API провайдера упал (500)**     | 3 повторные попытки. После 3-й — ошибка.                                                                                  |
| **Превышен лимит запросов (429)** | Пауза и повторная попытка. Если снова 429 — ошибка.                                                                       |
| **Неверный API-ключ (401/403)**   | Ошибка. Весь batch останавливается — повтор с невалидным ключом бессмыслен.                                               |
| **Баланс исчерпан (402)**         | Ошибка. Подсказка: «Пополните баланс или переключитесь на другого провайдера».                                            |
| **Промпт отклонён модерацией**    | Ошибка для этого запроса. Остальные запросы batch продолжаются.                                                           |
| **Диск заполнен**                 | Ошибка. Картинка не сохранена.                                                                                            |
| **Дубликат промпта**              | Режим SKIP: пропуск (бесплатно). Режим OVERWRITE: генерация заново.                                                       |
| **Провайдер вернул битый файл**   | Ошибка. Файл не сохраняется.                                                                                              |
| **Прокси недоступен**             | Ошибка. Автоматический переход на прямое подключение не происходит.                                                       |

### 10.2. Retry-стратегия

```
Попытка 1 → ошибка → пауза 2 сек
Попытка 2 → ошибка → пауза 4 сек
Попытка 3 → ошибка → FAILED

Повторы НЕ делаются при:
- 400 (ошибка промпта)
- 401/403 (неверный ключ)
- 402 (нет баланса)
```

> **⚠️ Узкое место:** Retry-стратегия использует фиксированную формулу `2^attempt` секунд
> без jitter (случайного смещения). При 429 (rate limit) от OpenAI повторная попытка
> через 2 секунды может снова попасть в лимит. Рекомендуется добавить jitter
> и учитывать заголовок `Retry-After` из ответа провайдера.
>
> Кроме того, **все 3 попытки** происходят внутри одного HTTP-запроса от клиента.
> Если 20 запросов и каждый делает 3 retry — клиент может ждать `20 × 3 × 8 = 480` сек
> только на повторах.

### 10.3. Восстановление при рестарте

При запуске сервера все «зависшие» запросы (статус RUNNING) автоматически переводятся в FAILED. Это гарантирует, что система не зависает после аварийного перезапуска.

## 12. Узкие места AI-интеграции (v1)

### 12.1. Rate limits провайдеров

AI-провайдеры ограничивают количество запросов в минуту:

| Провайдер    | Лимит (Tier 2)       | Что происходит при превышении                |
| ------------ | -------------------- | -------------------------------------------- |
| OpenAI       | ~15 RPM (images)     | 429 Too Many Requests                        |
| Stability AI | 150 запросов / 10 сек | 429 + lockout ~60 сек                       |

При batch 20 изображений на OpenAI (~15 RPM лимит) система начнёт получать 429
уже на 16-м запросе. Retry смягчает это, но **увеличивает общее время ожидания**.

**Решение (v2):** встроить rate limiter на стороне сервера (например, Guava RateLimiter
или Bucket4j), который сам ограничивает скорость отправки до лимита провайдера.

---

### 12.2. Буферизация изображений в памяти

Каждый вызов `generate()` возвращает `byte[]` (200–500 КБ).
Байты **не стримятся на диск** — они хранятся в памяти до сохранения.

| Batch        | Данных в памяти     |
| ------------ | ------------------- |
| 10 картинок  | 2–5 МБ              |
| 50 картинок  | 10–25 МБ            |
| 200 картинок | 40–100 МБ           |

Для текущих объёмов это некритично, но при параллельной генерации (несколько
пользователей одновременно) потребление может вырасти.

**Решение:** стримить байты сразу на диск (`InputStream → File`), не загружая
весь ответ в память.

## 14. FAQ

**Сколько стоит одна картинка?**
OpenAI: ~$0.02–0.19 (зависит от размера и качества). Stability AI: ~$0.03 (3 кредита).

**Какой провайдер работает из России без VPN?**
Stability AI работает напрямую. Для OpenAI нужен прокси.

**Что будет, если закончатся деньги?**
Система вернёт ошибку. Можно переключиться на другого провайдера или пополнить баланс.

**Что если отключат свет?**
Уже сгенерированные картинки сохранены. Незавершённые запросы при рестарте помечаются как ошибка. Повторная генерация — дубликаты пропускаются бесплатно (режим SKIP).

**Можно ли генерировать без интернета?**
Нет. AI-провайдеры работают в облаке.

**Картинки хранятся вечно?**
Да, пока не удалены вручную. Рекомендуется следить за размером диска.

**Как быстро генерируется?**
OpenAI: ~8–30 сек (зависит от модели). Stability: ~5–10 сек. Если 3 типа × 2 стиля = 6 картинок — около 1–3 минут (последовательно).

**Можно ли добавить другого провайдера?**
Да. Достаточно написать один Java-класс. Остальная система подхватит его автоматически.
