# 06. Прокси

## Зачем нужен

OpenAI **заблокирован** в России. Если сервер находится в РФ, запросы к `api.openai.com` не пройдут. Stability AI **работает напрямую** из России.

| Провайдер      | Из России              | Из EU / US  |
| -------------- | ---------------------- | ----------- |
| **OpenAI**     | Нужен прокси           | Напрямую    |
| **Stability**  | Напрямую               | Напрямую    |

```
Без прокси (сервер в РФ):
  Сервер ──► api.openai.com      → Заблокировано ✘
  Сервер ──► api.stability.ai    → Работает ✔

С прокси:
  Сервер ──► Прокси (IP за рубежом) ──► api.openai.com   → Работает ✔
  Сервер ──► Прокси (IP за рубежом) ──► api.stability.ai  → Работает ✔
```

---

## Принцип работы

Прокси настраивается **глобально** — один переключатель на весь сервер. Если включён, **все** провайдеры идут через прокси. Если выключен — все напрямую.

Каждый провайдер при старте в `@PostConstruct init()` создаёт один `RestTemplate` на весь жизненный цикл:

```java
if (props.getProxy().isEnabled() && !props.getProxy().getHost().isBlank()) {
    factory.setProxy(new Proxy(Proxy.Type.HTTP, new InetSocketAddress(host, port)));
}
```

- `proxy.enabled=true` → **оба** провайдера через прокси
- `proxy.enabled=false` → **оба** напрямую
- `RestTemplate` создаётся один раз — переключить без перезагрузки сервера **невозможно**

---

## Конфигурация

```properties
ai.proxy.enabled=true
ai.proxy.host=proxy-server.proxy-seller.io
ai.proxy.port=10000
ai.proxy.username=user123
ai.proxy.password=pass456
ai.proxy.type=HTTP
```

| Параметр   | Описание                                  | Обязателен            |
| ---------- | ----------------------------------------- | --------------------- |
| `enabled`  | Включить прокси для всех провайдеров      | да                    |
| `host`     | Адрес прокси-сервера                      | да (если enabled)     |
| `port`     | Порт                                      | да (если enabled)     |
| `username` | Логин для аутентификации                  | нет (см. ограничения) |
| `password` | Пароль                                    | нет (см. ограничения) |
| `type`     | Тип прокси: `HTTP` или `SOCKS5`          | нет (default `HTTP`)  |

---

## Управление прокси на клиенте

Переключатель прокси на фронтенде **отсутствует**. Это инфраструктурная настройка сервера, не пользовательская.

- Нужен прокси или нет — зависит от расположения сервера (Россия → нужен, за рубежом → нет)
- Прокси-креды — серверный секрет, на клиент не попадают
- Пользователь выбирает **провайдера**, а не способ маршрутизации

---

## Преимущества

1. **Простота настройки.** 5 строк в конфиге — один раз указал, все провайдеры ходят через прокси. Не нужно решать, кому прокси нужен, а кому нет
2. **Минимум кода.** Оба провайдера читают один и тот же `props.getProxy()`. Нет дублирования логики
3. **Быстрое подключение нового провайдера.** Добавляешь провайдер → он автоматически через прокси, ничего дополнительно не настраиваешь

---

## Недостатки

1. **Лишний hop для Stability.** Stability не заблокирован из РФ, но идёт через прокси — лишние +50–200 мс latency на каждый запрос
2. **Единая точка отказа.** Прокси упал → **оба** провайдера не работают, хотя Stability мог бы работать напрямую
3. **Расход трафика.** Stability возвращает raw bytes (~1–3 MB на картинку). Этот трафик идёт через прокси, расходуя bandwidth оплаченного канала
4. **Нет гранулярного отключения.** Если Stability начнёт глючить через прокси — нельзя отключить прокси только для него, не отключив OpenAI

---

## Известные ограничения

### Proxy auth не работает

`SimpleClientHttpRequestFactory` **не передаёт** `username` / `password` на прокси. Поля `ai.proxy.username` и `ai.proxy.password` в конфиге есть, но не используются при создании соединения.

Прокси работает только при авторизации **по IP** (whitelist IP сервера на стороне прокси-провайдера).

### Proxy type захардкожен

Код использует `Proxy.Type.HTTP` всегда, хотя в `AiProperties` есть поле `type` (может быть `SOCKS5`). Значение из конфига **игнорируется**.

```java
// Код сейчас:
factory.setProxy(new Proxy(Proxy.Type.HTTP, ...));

// Должно быть:
factory.setProxy(new Proxy(Proxy.Type.valueOf(props.getProxy().getType()), ...));
```

### Hot reload невозможен

`RestTemplate` создаётся в `@PostConstruct` один раз. Смена прокси-настроек = перезагрузка сервера.

### Фильтрация доменов

Если прокси настроен на whitelist доменов (только `api.openai.com`), запросы к `api.stability.ai` через него провалятся с 403.

## Эксплуатация

| Сценарий                        | Действие                                                            |
| ------------------------------- | ------------------------------------------------------------------- |
| Сервер в России, нужен OpenAI   | `ai.proxy.enabled=true`, указать host / port                        |
| Сервер за рубежом               | `ai.proxy.enabled=false`                                            |
| Прокси сменился                 | Поменять в `application.properties` → перезагрузить сервер          |
| Прокси упал                     | Retry (§7 в 01_Бэкенд). Если исчерпаны → FAILED. Пользователь может выбрать Stability |
| Нужен SOCKS5 вместо HTTP        | `ai.proxy.type=SOCKS5` — но тип захардкожен (см. ограничения)      |
| Прокси требует логин/пароль     | Авторизация по IP, креды из конфига не передаются (см. ограничения) |
