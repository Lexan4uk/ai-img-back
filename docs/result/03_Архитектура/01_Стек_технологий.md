# 01. Стек технологий

## Платформы

| Компонент | Платформа                                                    |
| --------- | ------------------------------------------------------------ |
| Сервер    | JVM (Java 21), Linux / Windows / macOS                       |
| Клиент    | JVM Desktop (Compose Multiplatform), Windows / Linux / macOS |
| СУБД      | PostgreSQL 15+                                               |

---

## Серверный стек

| Технология       | Версия | Назначение                                                         |
| ---------------- | ------ | ------------------------------------------------------------------ |
| Java             | 21     | Язык, LTS, text blocks, records, sealed classes                    |
| Spring Boot      | 3.5    | Веб-фреймворк, DI, автоконфигурация                                |
| Spring Data JDBC | —      | Data-access (NamedParameterJdbcTemplate)                           |
| PostgreSQL       | 15+    | Реляционная СУБД (BIGSERIAL PK)                                    |
| Liquibase        | —      | Миграции БД (XML changesets)                                       |
| Lombok           | —      | Boilerplate reduction (@Getter, @Setter, @RequiredArgsConstructor) |
| OkHttp           | 4.12   | HTTP-клиент (в модуле client-utils)                                |
| Jackson          | —      | JSON-сериализация (ObjectMapper, JavaTimeModule)                   |
| Maven            | —      | Сборка, multi-module (parent + client-utils + web)                 |
| Testcontainers   | —      | Интеграционные тесты с реальным PostgreSQL                         |
| JUnit 5          | —      | Фреймворк тестирования                                             |
| RestTemplate     | —      | HTTP-клиент для AI-провайдеров (OpenAI, Stability AI)              |

### Обоснование выбора

- **Spring Boot + Data JDBC** вместо JPA: проект использует чистый SQL
  через `NamedParameterJdbcTemplate`. Это даёт полный контроль над запросами,
  простоту отладки и отсутствие магии ORM (lazy loading, N+1).

- **Liquibase** вместо Flyway: XML-формат, декларативный подход,
  возможность rollback.

- **Multi-module Maven**: модуль `client-utils` содержит общие DTO, интерфейсы
  контроллеров и HTTP-клиент. Этот модуль подключается и к серверу (web),
  и к десктоп-клиенту — **единый контракт API**.

- **BIGSERIAL** вместо UUID: упрощённые PK, меньший размер индексов,
  автоинкремент, удобство при отладке.

## Клиентский стек

| Технология            | Версия         | Назначение                                   |
| --------------------- | -------------- | -------------------------------------------- |
| Kotlin                | 2.1+           | Язык (JVM target)                            |
| Compose Multiplatform | 1.7+           | UI-фреймворк (Desktop)                       |
| Material 3            | —              | Дизайн-система (тема, компоненты)            |
| client-utils          | 0.0.1-SNAPSHOT | Общие DTO + ClientUtil (HTTP-клиент)         |
| OkHttp                | 4.12           | HTTP-клиент (транзитивно через client-utils) |
| Jackson               | 2.18           | JSON-парсинг (jackson-databind + jsr310)     |
| Gradle                | 8.x            | Система сборки                               |

### Обоснование выбора

- **Compose Multiplatform Desktop** (не Android): приложение — десктопный клиент
  для управления AI-генерацией. Compose даёт декларативный UI, Material 3,
  и возможность при необходимости портировать на Android.

- **client-utils как Maven-артефакт**: клиент подключает тот же модуль
  `ai-img-client-utils`, что и сервер. Интерфейсы контроллеров (`IImageTypeController`,
  `IStyleController` и др.) используются как прокси через `ClientUtil` — вызов
  метода на интерфейсе автоматически становится HTTP-запросом.

- **Kotlin + Java interop**: client-utils написан на Java, клиент — на Kotlin.
  Совместимость обеспечивается JVM bytecode.

## Нефункциональные требования

| Параметр                      | Значение                                               |
| ----------------------------- | ------------------------------------------------------ |
| Время запуска сервера         | < 5 сек                                                |
| Время отклика API (CRUD)      | < 200 мс                                               |
| Время отклика API (генерация) | Зависит от AI-провайдера                               |
| Максимум ассетов в галерее    | 10 000+ (серверная пагинация при необходимости)        |
| Поддержка тем                 | Светлая / тёмная (Material 3)                          |
| Конкурентные пользователи     | Справочники глобальные, ассеты публичные               |
| Тестовое покрытие             | Интеграционные тесты на все эндпоинты (Testcontainers) |
