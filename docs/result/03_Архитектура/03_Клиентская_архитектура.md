# 03. Клиентская архитектура

## Общие сведения

Клиент — десктопное приложение на **Compose Multiplatform** (JVM target).
Собирается Gradle, использует модуль `client-utils` из серверного проекта
как Maven-зависимость.

**Репозиторий:** `D:\vse\AndroidStudioProjects\aiimageclient`

## Текущее состояние

На данный момент клиент содержит минимальный каркас:

```
aiimageclient/
├── composeApp/
│   ├── build.gradle.kts            ← Зависимости (client-utils, OkHttp, Jackson)
│   └── src/jvmMain/kotlin/
│       └── com.example.ai_image_client/
│           ├── main.kt             ← Точка входа (Window, App)
│           ├── App.kt              ← Тестовая кнопка
│           ├── Platform.kt         ← Platform.name = "Java ${version}"
│           ├── Greeting.kt         ← Тестовый greet()
│           └── core/
│               └── ServerConnectionManager.kt  ← Обёртка ClientUtil
├── gradle/
├── settings.gradle.kts
└── build.gradle.kts                ← Root build (plugins)
```

`ServerConnectionManager` оборачивает `ClientUtil` из client-utils:

```kotlin
class ServerConnectionManager(
    private var serverUrl: String,
    private var userId: Long
) {
    @Volatile
    private var clientUtil: ClientUtil = ClientUtil(serverUrl, userId)

    fun <T> getController(controllerClass: Class<T>): T =
        clientUtil.getClient(controllerClass)

    fun updateServerUrl(newUrl: String) { ... }
    fun updateUserId(newUserId: Long) { ... }
}
```

## Архитектурный паттерн

Используется **MVVM** (Model-View-ViewModel):

```
┌───────────────────────────────────────┐
│              View (Compose)            │
│  @Composable функции                  │
│  Рисуют UI по состоянию ViewModel     │
├───────────────────────────────────────┤
│            ViewModel                   │
│  Хранит MutableState / StateFlow      │
│  Вызывает методы контроллеров         │
│  Обрабатывает ошибки                  │
├───────────────────────────────────────┤
│              Model                     │
│  ClientUtil → HTTP-прокси             │
│  DTO из client-utils                  │
│  Сервер отвечает за бизнес-логику     │
└───────────────────────────────────────┘
```

### Управление состоянием

- **Compose State** (`mutableStateOf`, `mutableStateListOf`) — для UI-состояния
- **StateFlow** (Kotlin coroutines) — для асинхронных операций (сетевые запросы)
- **remember / rememberSaveable** — кеширование в рамках композиции

### Пример (GalleryViewModel):

```kotlin
class GalleryViewModel(
    private val connectionManager: ServerConnectionManager
) {
    var types by mutableStateOf<List<ImageTypeDTO>>(emptyList())
        private set

    var selectedType by mutableStateOf<ImageTypeDTO?>(null)
        private set

    var assets by mutableStateOf<List<AssetDTO>>(emptyList())
        private set

    var isLoading by mutableStateOf(false)
        private set

    fun loadTypes() {
        isLoading = true
        // В coroutine scope:
        val ctrl = connectionManager.getController(IImageTypeController::class.java)
        types = ctrl.getAll()
        isLoading = false
    }

    fun selectType(type: ImageTypeDTO) {
        selectedType = type
        val ctrl = connectionManager.getController(IAssetController::class.java)
        assets = ctrl.getByImageType(type.id)
    }
}
```

## Навигация

Навигация реализуется через **sealed class** и `when`-expression:

```kotlin
sealed class Screen {
    data object Gallery : Screen()
    data object Generation : Screen()
}

@Composable
fun AppContent(currentScreen: Screen, ...) {
    Row {
        Sidebar(currentScreen, onNavigate = { ... })

        when (currentScreen) {
            Screen.Gallery    -> GalleryScreen(...)
            Screen.Generation -> GenerationScreen(...)
        }
    }
}
```

Sidebar содержит 2 пункта: Галерея и Генерация.
Состояние (свёрнута/развёрнута) сохраняется в `PreferencesManager`.

## Темы

Material 3 с двумя вариантами:

```kotlin
@Composable
fun AppTheme(isDark: Boolean, content: @Composable () -> Unit) {
    val colorScheme = if (isDark) darkColorScheme(
        background = Color(0xFF1E1E1E),
        surface = Color(0xFF252525),
        primary = Color(0xFF64B5F6),
        onBackground = Color(0xFFE0E0E0),
    ) else lightColorScheme(
        background = Color(0xFFFFFFFF),
        surface = Color(0xFFF5F5F5),
        primary = Color(0xFF1976D2),
        onBackground = Color(0xFF212121),
    )

    MaterialTheme(colorScheme = colorScheme, content = content)
}
```

Переключение: кнопка в Header. Выбор сохраняется в `PreferencesManager`.

## Зависимости (build.gradle.kts)

```kotlin
jvmMain.dependencies {
    // Общий контракт API (DTO + ClientUtil + интерфейсы контроллеров)
    implementation("com.example:ai-img-client-utils:0.0.1-SNAPSHOT")

    // HTTP-клиент (нужен для ClientUtil)
    implementation("com.squareup.okhttp3:okhttp:4.12.0")

    // JSON (Jackson) — десериализация DTO
    implementation("com.fasterxml.jackson.core:jackson-databind:2.18.3")
    implementation("com.fasterxml.jackson.datatype:jackson-datatype-jsr310:2.18.3")

    // Compose Desktop
    implementation(compose.desktop.currentOs)
    implementation(compose.material3)
}
```
