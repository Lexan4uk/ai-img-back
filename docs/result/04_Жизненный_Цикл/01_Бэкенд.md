# Бэкенд — серверная валидация и обработка

Справочник серверной логики: какие проверки делает сервер, что пропускает, как обрабатывает запросы и где хранит данные.

---

## Цикл обработки запросов

### Создание типа / стиля

1. Клиент отправляет `POST /image-types` или `POST /styles` с заголовком `UserId`
2. Сервис тримит name, проверяет not null / blank → 400
3. INSERT в БД → проверка UNIQUE LOWER(name) и FK userId
4. Ответ: созданная запись с id

### Удаление типа / стиля

1. Клиент отправляет `DELETE /image-types/{id}` или `DELETE /styles/{id}` с заголовком `UserId`
2. Проверки: запись не найдена → 404, id = 1 → 403, userId ≠ создатель → 403
3. Переназначение assets и requests на id=1 (обход RESTRICT)
4. DELETE из БД

### Избранное

1. Toggle: `POST /{id}/favorite` или `DELETE /{id}/favorite`, заголовок `UserId`
2. POST: проверка существования записи → 404; INSERT ON CONFLICT DO NOTHING
3. DELETE: без проверок → тихий 200

### Генерация

1. `POST /generations/check` → подсчёт дубликатов по hash (подробнее → §3)
2. Клиент решает: пропустить дубликаты или пересоздать
3. `POST /generations` → создание batch → формирование requests
4. Цикл обработки — для каждой пары (тип, стиль):
   - Сборка `finalPrompt` из шаблона (`{type}` + `{style}` + `{prompt}`)
   - Выбор провайдера через `ProviderRouter` (указанный или дефолтный)
   - HTTP-запрос к AI-провайдеру (OpenAI: JSON → base64, Stability: multipart → raw bytes)
   - Если `ai.proxy.enabled=true` — запрос идёт через прокси-сервер (см. [06_Прокси.md](../../03_Архитектура/06_Прокси.md))
   - При ошибке — retry с экспоненциальной задержкой (см. [05*Интеграция*с_AI.md](../../03_Архитектура/05_Интеграция_с_AI.md), §12)
   - Провайдер возвращает `byte[]` → валидация magic bytes → сохранение на диск → создание asset в БД
5. Ответ: список результатов (DONE / FAILED) по каждой паре

### Получение данных

1. `GET /image-types`, `GET /styles` — полный список, без пагинации
2. `GET /assets?imageTypeId=X&styleId=Y&page=0&size=20` — с пагинацией
3. `GET /*/favorites` — списки избранного пользователя
4. Валидации нет ни на одном из эндпоинтов

## 1. Тип изображения

### `name`

| Факт                  | Детали                                             |
| --------------------- | -------------------------------------------------- |
| Сервис тримит         | `"  foo  "` → `"foo"`                              |
| Проверка null/blank   | после trim → 400                                   |
| UNIQUE по LOWER(name) | дубликат (регистронезависимый) → 400 DataIntegrity |
| Длина не проверяется  | клиент лимитирует 100 символов (§1 фронтенда)      |
| Seed-запись id=1      | «Неопределённый», защищена от удаления             |

При ошибке UNIQUE — ответ `400 DataIntegrityViolation` без указания конкретного поля.

### `typePrompt`

| Факт                      | Детали                                          |
| ------------------------- | ----------------------------------------------- |
| null/blank не проверяется | клиент блокирует пустой промпт (§1 фронтенда)   |
| Длина не проверяется      | клиент лимитирует 1 000 символов (§1 фронтенда) |
| Nullable в БД             | да                                              |

### `created_by_user_id`

| Факт                 | Детали                                               |
| -------------------- | ---------------------------------------------------- |
| Существование userId | не проверяется сервисом, ловится FK при INSERT       |
| FK → users(id)       | невалидный userId → 400 DataIntegrity                |
| ON DELETE SET NULL   | при удалении пользователя — тип остаётся, автор null |

## 3. Генерация

### `userPrompt`

| Факт                 | Детали                                          |
| -------------------- | ----------------------------------------------- |
| Длина не проверяется | клиент лимитирует 2 000 символов (§3 фронтенда) |
| Тримится             | перед сохранением                               |
| Проверка null/blank  | → 400 «Промпт не может быть пустым»             |

### `imageTypeIds` и `styleIds`

| Факт                     | Детали                                                    |
| ------------------------ | --------------------------------------------------------- |
| Размер списков           | не проверяется (клиент по 10, §3 фронтенда)               |
| Дубликаты id в списках   | не проверяются (клиент не может дважбы выбрать 1 чекбокс) |
| null / пустой            | → 400 «Выберите хотя бы один тип/стиль»                   |
| Содержит id=1            | → 400 «Нельзя генерировать с неопределённым…»             |
| Существование каждого id | getById → 404 если не найден                              |

### Провайдер и модель

| Факт           | Детали                                                  |
| -------------- | ------------------------------------------------------- |
| provider null  | → дефолт из `aiProperties.defaultProvider` (`"openai"`) |
| model          | передаётся провайдеру; Stability подставляет в URL (§7) |
| Проверка имени | ProviderRouter.resolve — **после** создания batch       |

### `generationParams`

| Факт            | Детали                                                                            |
| --------------- | --------------------------------------------------------------------------------- |
| null            | → `"{}"`                                                                          |
| Формат          | JSON-строка `{"aspectRatio": "1:1"}`, задается клиентом из подготовленного списка |
| Валидность JSON | не проверяется — ошибка при `CAST AS jsonb` в БД                                  |

### Дедупликация

| Факт                       | Детали                                                   |
| -------------------------- | -------------------------------------------------------- |
| Двухшаговый процесс        | `POST /generations/check` → диалог → `POST /generations` |
| Хэш                        | SHA-256 от `finalPrompt + "\|" + generationParams`       |
| existsByHash               | ищет в `generation_requests` WHERE `status = 'DONE'`     |
| overwriteDuplicates: false | дубликат → request **не создаётся** (`continue`)         |
| overwriteDuplicates: true  | генерация для всех пар заново                            |
| Все пары — дубликаты       | пустой список requests, генерация не запускается         |

### Цикл обработки request-ов

Для каждого request в batch:

1. `authFailed?` → markFailed(«ошибка авторизации»), пропуск
2. Статус → `RUNNING`
3. `composeFinalPrompt(typePrompt, stylePrompt, userPrompt)`
4. `provider.generate(finalPrompt, aspectRatio, model)` → `byte[]`
5. `imageStorageService.save(byte[])` — валидация magic bytes (PNG/JPEG/WebP) + size ≥ 1 KB → сохранение как `UUID.ext`
6. `assetRepository.create(imageTypeId, styleId, fileUri)` → asset
7. Статус → `DONE`, привязка `created_asset_id`

### Ошибки в цикле (шаги 4–6)

| Ситуация                        | Результат                                                   |
| ------------------------------- | ----------------------------------------------------------- |
| Retryable `AiProviderException` | retry с экспоненциальной задержкой, при исчерпании → FAILED |
| 401 / 402 / 403                 | FAILED + `authFailed=true` → все следующие тоже FAILED      |
| Non-retryable                   | сразу FAILED                                                |
| `IOException` (диск)            | FAILED                                                      |
| Неожиданная `Exception`         | FAILED                                                      |

Ответ клиенту: `List<GenerationResult>` с финальными статусами (`DONE` / `FAILED`).

## 7. Провайдеры

### OpenAI

| Параметр            | Значение                                       |
| ------------------- | ---------------------------------------------- |
| Endpoint            | `https://api.openai.com/v1/images/generations` |
| Модель по умолчанию | `gpt-image-1.5`                                |
| n                   | фиксировано `1`                                |
| quality             | `"high"`                                       |
| output_format       | `"png"`                                        |
| Таймаут             | 90 сек                                         |
| Макс. попыток       | 3                                              |
| Retry delay         | `2^attempt × 1000` мс                          |

**Маппинг aspect ratio → size:**

| aspectRatio   | size          |
| ------------- | ------------- |
| `"1:1"`       | `"1024x1024"` |
| `"1:1.5"`     | `"1024x1536"` |
| `"1.5:1"`     | `"1536x1024"` |
| другое / null | `"auto"`      |

### Stability

| Параметр         | Значение                                                   |
| ---------------- | ---------------------------------------------------------- |
| URL              | `ai.stability.url` + `"/"` + `model` → `.../generate/core` |
| API формат       | multipart/form-data                                        |
| Стоимость        | 3 кредита                                                  |
| Разрешение       | ~1.5 мегапикселя                                           |
| Таймаут          | 90 сек                                                     |
| Макс. попыток    | 2                                                          |
| Параметр `model` | подставляется в URL (default `core` из конфига)            |

**Допустимые aspect_ratio:**
`16:9`, `1:1`, `21:9`, `2:3`, `3:2`, `4:5`, `5:4`, `9:16`, `9:21`

- Ratio не из списка → выбирается ближайшее по числовому соотношению.
- Ratio null / blank → fallback `"1:1"`.

### ProviderRouter

| Ситуация                    | Результат                           |
| --------------------------- | ----------------------------------- |
| Провайдер указан, ключ есть | ОК                                  |
| Провайдер не в реестре      | 500: «Неизвестный провайдер»        |
| Провайдер без ключа         | 500: «не настроен (нет API-ключа)»  |
| null / blank / `"auto"`     | дефолт → первый доступный           |
| Ни один не доступен         | 500: «Нет доступных AI-провайдеров» |

Поиск провайдера — case-insensitive.

## 8. Обработчик ошибок

| Исключение                        | HTTP | Тело                                                             |
| --------------------------------- | ---- | ---------------------------------------------------------------- |
| `EntityNotFoundException`         | 404  | `{"error": "сообщение"}`                                         |
| `IllegalArgumentException`        | 400  | `{"error": "сообщение"}`                                         |
| `DataIntegrityViolationException` | 400  | `{"error": "Нарушение ограничений базы данных"}` — детали скрыты |
| `IllegalStateException`           | 403  | `{"error": "сообщение"}`                                         |
| `AiProviderException`             | 502  | `{"error": "Ошибка AI-провайдера: сообщение"}`                   |
| `Exception` (всё остальное)       | 500  | `{"error": "сообщение исключения"}` — утечка деталей             |

`AiProviderException` имеет отдельный обработчик → 502 Bad Gateway. В штатном потоке она перехватывается внутри `GenerationService` и не пробрасывается до контроллера, обработчик добавлен как защитная сетка.

| --------------- | ----------- | -------- | ------- | ------------------------------ |

| `image_type_id`     | BIGINT      | да       | —           | —                                                | `image_types(id)` **RESTRICT**                 |
| `style_id`          | BIGINT      | да       | —           | —                                                | `styles(id)` **RESTRICT**                      |
| `final_prompt_hash` | TEXT        | да       | —           | —                                                | —                                              |

## 10. Сводка пробелов валидации

| Пробел              | Где                       | Последствие                                 | Клиент закрывает?                               | Важность |
| ------------------- | ------------------------- | ------------------------------------------- | ----------------------------------------------- | -------- |
| Длина строк         | Все сервисы               | TEXT без лимита — мегабайты данных          | да (100 / 1 000 / 2 000 символов)               | —        |
| Размер списков      | GenerationService         | imageTypeIds × styleIds = N² request-ов     | да (по 10 = макс 100)                           | —        |
| Дубликаты в списках | GenerationService         | Дублирующиеся generation request-ы          | да (невозможно несколько раз выбрать 1 чекбокс) | —        |
| JSON-валидация      | GenerationService         | generationParams невалидный → ошибка БД     | нет (формируется программно)                    | —        |
| Имя провайдера      | GenerationService         | Проверяется после создания batch            | нет (формируется программно)                    | —        |
| Пагинация           | `/image-types`, `/styles` | Полный список без лимита                    | нет (десятки записей — допустимо)               | —        |
| AiProviderException | GlobalExceptionHandler    | Обработчик → 502 Bad Gateway                | да                                              | —        |
| UserId заголовок    | Все контроллеры           | Не проверяется существование пользователя   | нет                                             | средняя  |
| Утечка ошибок       | GlobalExceptionHandler    | Exception → 500 с сырым сообщением          | нет                                             | средняя  |

**Пояснение к важности:**

- **Средняя (UserId)** — любой Long в заголовке `UserId` принимается без проверки. Несуществующий userId = данные без владельца. Чужой userId = доступ к чужим избранным и типам.
- **Средняя (утечка ошибок)** — `Exception.getMessage()` попадает в JSON-ответ. Может содержать имена таблиц, SQL-фрагменты, пути файловой системы. Не критично для внутреннего использования, но проблема при публичном деплое.
