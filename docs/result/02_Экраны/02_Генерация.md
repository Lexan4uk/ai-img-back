# S-02. Генерация

## Назначение

Запуск генерации изображений через AI. Пользователь выбирает типы и стили
(множественный выбор), вводит промпт и параметры, запускает процесс.

Система формирует декартово произведение (типы × стили) и последовательно генерирует.

## Блок 1: Выбор типов (множественный)

**Расположение:** верхняя левая область

| Элемент   | Описание                                                     |
| --------- | ------------------------------------------------------------ |
| Поиск     | Фильтрация по имени (case-insensitive подстрока)             |
| Чекбоксы  | Множественный выбор типов                                    |
| ★ / ☆     | Toggle избранного                                            |
| 🗑        | Кнопка удаления (видна только у типов текущего пользователя) |
| Счётчик   | «Выбрано: N» внизу списка                                    |
| + Создать | Кнопка создания нового типа (внизу списка, на всю ширину)    |
| Секции    | Избранные сверху, остальные ниже (без дублирования)          |

**Ограничения:**

- «Неопределённый» (id=1) **не отображается** — генерация с ним запрещена.
- Минимум 1 тип должен быть выбран для генерации.
- Максимум 10 типов. При достижении лимита остальные чекбоксы блокируются, показывается подсказка «максимум 10».

**Данные:**

- `GET /image-types` → все типы (кроме id=1)
- `GET /image-types/favorites` → избранные

---

## Блок 2: Выбор стилей (множественный)

Зеркально блоку 1, но для стилей.

- «Неопределённый» (id=1) **не отображается**.
- Минимум 1 стиль должен быть выбран.
- Максимум 10 стилей (аналогично типам).

**Данные:**

- `GET /styles` → все стили (кроме id=1)
- `GET /styles/favorites` → избранные

---

## Блок 3: Промпт и параметры

**Расположение:** нижняя часть экрана (на всю ширину)

| Элемент                | Описание                                                                     |
| ---------------------- | ---------------------------------------------------------------------------- |
| Промпт                 | Многострочное текстовое поле (3–5 строк), обязательное. Лимит 2 000 символов |
| Модель                 | Dropdown выбора AI-модели (OpenAI GPT Image 1 Mini / 1.5, Stability Core)    |
| Aspect Ratio           | Dropdown с допустимыми соотношениями сторон (зависит от выбранной модели)    |
| Калькулятор            | «Будет сгенерировано: N изображений» (обновляется в реальном времени)        |
| Кнопка «Сгенерировать» | Запуск генерации (на всю ширину, disabled пока форма невалидна)              |

При смене модели список aspect ratio обновляется. Если текущее значение не поддерживается новой моделью — сбрасывается на значение по умолчанию.

## Состояния экрана

### До генерации

- Кнопка «Сгенерировать» активна если: промпт не пустой + ≥1 тип + ≥1 стиль.
- Калькулятор показывает: типы × стили = N.
- Максимум запросов: 10 × 10 = 100.

### Проверка дубликатов

При нажатии «Сгенерировать» выполняется двухшаговый процесс:

1. `POST /generations/check` — сервер подсчитывает дубликаты (пары тип × стиль, для которых ассет уже существует).
2. Если дубликаты есть — отображается диалог:

```
┌─── Найдены дубликаты ──────────────────────────────┐
│                                                      │
│  Всего: 6, из них дубликатов: 2, новых: 4            │
│                                                      │
│  [ Пропустить дубликаты ]   [ Пересоздать все ]      │
└──────────────────────────────────────────────────────┘
```

- **Пропустить дубликаты** → `POST /generations` с `overwriteDuplicates: false`. Сервер не создаёт запросы для дубликатов — они не попадают в ответ.
- **Пересоздать все** → `POST /generations` с `overwriteDuplicates: true`. Генерация выполняется для всех пар заново.

3. Если дубликатов нет — генерация запускается сразу, без диалога.

### Во время генерации

```
┌─── Генерация ──────────────────────────────────────────┐
│                                                          │
│              ⏳ Генерация...                             │
│                                                          │
│  Ожидание ответа сервера.                               │
│  Будет сгенерировано: 6 изображений                     │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

- Кнопка «Сгенерировать» блокируется (disabled) на время запроса.
- Отображается спиннер.
- `POST /generations` — синхронный вызов: клиент ждёт полный ответ.
- Нет прогресс-бара — пользователь не знает, сколько картинок готово.

### После генерации

```
┌─── Результаты ──────────────────────────────────────────┐
│                                                          │
│  Готово! 5 из 6 успешно, 1 ошибка                       │
│                                                          │
│  ✅ Фотография × Айвазовский — создан ассет #142        │
│  ✅ Фотография × Warcraft — создан ассет #143           │
│  ✅ Фотография × Минимализм — создан ассет #144         │
│  ✅ Картина × Айвазовский — создан ассет #145           │
│  ✅ Картина × Warcraft — создан ассет #146              │
│  ❌ Картина × Минимализм — ОШИБКА: rate limit           │
│                                                          │
│  [ Открыть в галерее ]    [ Новая генерация ]            │
└──────────────────────────────────────────────────────────┘
```

Каждая пара показывается со статусом: DONE (✅) или FAILED (❌).

При `overwriteDuplicates: false` дубликаты не создают записей и не отображаются в результатах.

- «Открыть в галерее» → переход на S-01 с предвыбором типа.
- «Новая генерация» → сброс формы.

### При ошибках AI (часть запросов упала)

Если отдельные AI-вызовы вернули ошибку — остальные продолжаются.
Результаты отображаются поштучно:

```

│  ❌ Фотография × Минимализм — ОШИБКА: rate limit        │
│  ✅ Картина × Айвазовский — создан ассет #145           │
│  ✅ Картина × Warcraft — создан ассет #146              │

│                                                          │
```

### При полном сбое (сервер упал / сеть разорвалась)

Если HTTP-запрос полностью провалился (таймаут, connection refused, crash сервера),
клиент создаёт **синтетические результаты** — по одному FAILED на каждую
запланированную пару тип × стиль. Все показываются как ❌:

```
┌─── Результаты ──────────────────────────────────────────┐
│                                                          │
│  Готово! 0 из 6 успешно, 6 ошибок                       │
│                                                          │
│  ❌ Фотография × Айвазовский — ОШИБКА: Connection refused│
│  ❌ Фотография × Warcraft — ОШИБКА: Connection refused   │
│  ❌ Фотография × Минимализм — ОШИБКА: Connection refused │
│  ❌ Картина × Айвазовский — ОШИБКА: Connection refused   │
│  ❌ Картина × Warcraft — ОШИБКА: Connection refused      │
│  ❌ Картина × Минимализм — ОШИБКА: Connection refused    │
│                                                          │
│  [ Открыть в галерее ]    [ Новая генерация ]            │
└──────────────────────────────────────────────────────────┘
```

> Пользователь всегда видит итог по каждой паре — сколько успешно, сколько нет.

### Ошибки создания типа/стиля

Если при создании нового типа или стиля сервер вернул ошибку —
отображается **AlertDialog** с текстом ошибки и кнопкой «OK».

## Управление справочниками (встроенное)

На этом же экране пользователь может **создавать** и **удалять** типы/стили:

### Создание

Кнопка `[+ Создать]` под соответствующим списком.
Открывает диалог:

```
┌─── Новый тип ────────────────┐
│ Название:  [____________]     │
│ Промпт:   [____________]     │
│                               │
│ [ Отмена ]   [ Создать ]      │
└───────────────────────────────┘
```

Ограничения полей:

- Название: обязательное, до 100 символов.
- Промпт: обязательный, до 1 000 символов.
- Кнопка «Создать» неактивна, пока оба поля не заполнены.

→ `POST /image-types` или `POST /styles`

### Удаление

Кнопка 🗑 рядом с типом/стилем (видна только у элементов текущего пользователя).
Подтверждение: «Удалить тип "Фото"? Все ассеты будут переназначены на "Неопределённый"».

→ `DELETE /image-types/{id}` или `DELETE /styles/{id}`

## Валидация

| Ситуация          | Поведение                                       |
| ----------------- | ----------------------------------------------- |
| Промпт пустой     | Кнопка «Сгенерировать» неактивна                |
| 0 типов выбрано   | Кнопка неактивна                                |
| 0 стилей выбрано  | Кнопка неактивна                                |
| >10 типов/стилей  | Чекбоксы заблокированы, подсказка «максимум 10» |
| Сервер вернул 400 | Snackbar с текстом ошибки                       |
| Сервер вернул 404 | «Тип/стиль не найден — возможно удалён»         |

## Данные (API)

| Действие            | Запрос                       |
| ------------------- | ---------------------------- |
| Загрузить типы      | `GET /image-types`           |
| Загрузить стили     | `GET /styles`                |
| Избранные типы      | `GET /image-types/favorites` |
| Избранные стили     | `GET /styles/favorites`      |
| Создать тип         | `POST /image-types`          |
| Создать стиль       | `POST /styles`               |
| Удалить тип         | `DELETE /image-types/{id}`   |
| Удалить стиль       | `DELETE /styles/{id}`        |
| Toggle избранное    | `POST/DELETE .../favorite`   |
| Проверка дубликатов | `POST /generations/check`    |
| Запуск генерации    | `POST /generations`          |
