# S-02. Генерация

## Назначение

Запуск генерации изображений через AI. Пользователь выбирает типы и стили
(множественный выбор), вводит промпт и параметры, запускает процесс.

Система формирует декартово произведение (типы × стили) и последовательно генерирует.

## Блок 1: Выбор типов (множественный)

**Расположение:** верхняя левая область

| Элемент  | Описание                         |
| -------- | -------------------------------- |
| Поиск    | Фильтрация по имени              |
| Чекбоксы | Множественный выбор типов        |
| Счётчик  | «Выбрано: N»                     |
| ★ / ☆    | Toggle избранного                |
| Секции   | Избранные сверху, остальные ниже (без дублирования) |

**Ограничения:**

- «Неопределённый» (id=1) **не отображается** — генерация с ним запрещена.
- Минимум 1 тип должен быть выбран для генерации.

**Данные:**

- `GET /image-types` → все типы (кроме id=1)
- `GET /image-types/favorites` → избранные

---

## Блок 2: Выбор стилей (множественный)

Зеркально блоку 1, но для стилей.

- «Неопределённый» (id=1) **не отображается**.
- Минимум 1 стиль должен быть выбран.

**Данные:**

- `GET /styles` → все стили (кроме id=1)
- `GET /styles/favorites` → избранные

---

## Блок 3: Промпт и параметры

**Расположение:** нижняя часть экрана (на всю ширину)

| Элемент                | Описание                                                    |
| ---------------------- | ----------------------------------------------------------- |
| Промпт                 | Многострочное текстовое поле. @NotBlank                     |
| Ширина / Высота        | Два числовых поля (по умолчанию 1024)                       |
| Дубликаты              | Radio: SKIP (по умолчанию) / OVERWRITE                      |
| Калькулятор            | «Будет сгенерировано: N типов × M стилей = N×M изображений» |
| Кнопка «Сгенерировать» | Запуск генерации                                            |

## Состояния экрана

### До генерации

- Кнопка «Сгенерировать» активна если: промпт не пустой + ≥1 тип + ≥1 стиль.
- Калькулятор обновляется в реальном времени.

### Во время генерации

```
┌─── Генерация ──────────────────────────────────────────┐
│                                                          │
│              ⏳ Генерация...                             │
│                                                          │
│  Ожидание ответа сервера.                               │
│  Будет сгенерировано: 6 изображений                     │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

- Кнопка «Сгенерировать» блокируется (disabled) на время запроса.
- Отображается спиннер/крутилка.
- `POST /generations` — синхронный вызов: клиент ждёт полный ответ.

### После генерации

```
┌─── Результаты ──────────────────────────────────────────┐
│                                                          │
│  Готово! 5 из 6 успешно, 1 пропущен (дубликат)          │
│                                                          │
│  ✅ Фотография × Айвазовский — создан ассет #142        │
│  ✅ Фотография × Warcraft — создан ассет #143           │
│  ✅ Фотография × Минимализм — создан ассет #144         │
│  ✅ Картина × Айвазовский — создан ассет #145           │
│  ✅ Картина × Warcraft — создан ассет #146              │
```

- «Открыть в галерее» → переход на S-01 с предвыбором типа.
- «Новая генерация» → сброс формы.

## Управление справочниками (встроенное)

На этом же экране пользователь может **создавать** и **удалять** типы/стили:

### Создание

Кнопка `[+ Тип]` / `[+ Стиль]` под соответствующим списком.
Открывает inline-форму или диалог:

```
┌─── Новый тип ────────────────┐
│ Название:  [____________]     │
│ Промпт:   [____________]     │
│                               │
│ [ Отмена ]   [ Создать ]      │
└───────────────────────────────┘
```

→ `POST /image-types` или `POST /styles`

### Удаление

Кнопка 🗑 рядом с типом/стилем (видна только создателю).
Подтверждение: «Удалить тип "Фото"? Все ассеты будут переназначены на "Неопределённый"».

→ `DELETE /image-types/{id}` или `DELETE /styles/{id}`

## Валидация

| Ситуация          | Поведение                               |
| ----------------- | --------------------------------------- |
| Промпт пустой     | Кнопка «Сгенерировать» неактивна        |
| 0 типов выбрано   | Кнопка неактивна                        |
| 0 стилей выбрано  | Кнопка неактивна                        |
| Сервер вернул 400 | Snackbar с текстом ошибки               |
| Сервер вернул 404 | «Тип/стиль не найден — возможно удалён» |

## Данные (API)

| Действие         | Запрос                       |
| ---------------- | ---------------------------- |
| Загрузить типы   | `GET /image-types`           |
| Загрузить стили  | `GET /styles`                |
| Избранные типы   | `GET /image-types/favorites` |
| Избранные стили  | `GET /styles/favorites`      |
| Создать тип      | `POST /image-types`          |
| Создать стиль    | `POST /styles`               |
| Удалить тип      | `DELETE /image-types/{id}`   |
| Удалить стиль    | `DELETE /styles/{id}`        |
| Toggle избранное | `POST/DELETE .../favorite`   |
| Запуск генерации | `POST /generations`          |
