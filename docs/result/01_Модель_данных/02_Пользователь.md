# 02. Пользователь (users)

## Назначение

Представляет зарегистрированного пользователя системы.
Используется как владелец сгенерированных изображений, создатель типов/стилей,
субъект избранного.

> **Текущая аутентификация** — заглушка: header `UserId: <Long>`.
> В продакшене будет заменено на JWT / OAuth2.

---

## Структура таблицы

```sql
CREATE TABLE users (
    id           BIGSERIAL  PRIMARY KEY,
    email        TEXT       NOT NULL,
    display_name TEXT       NOT NULL,
    CONSTRAINT uq_users_email UNIQUE (email)
);
```

| Поле           | Тип       | Обязательно | Описание                                           |
| -------------- | --------- | ----------- | -------------------------------------------------- |
| `id`           | BIGSERIAL | auto        | Уникальный идентификатор                           |
| `email`        | TEXT      | ✅          | Email (уникальный, используется для идентификации) |
| `display_name` | TEXT      | ✅          | Отображаемое имя пользователя                      |

---

## API

| Метод  | URL           | UserId | Body                                       | Описание             |
| ------ | ------------- | ------ | ------------------------------------------ | -------------------- |
| POST   | `/users`      | —      | `{ "email": "...", "displayName": "..." }` | Создать пользователя |
| GET    | `/users`      | —      | —                                          | Получить всех        |
| GET    | `/users/{id}` | —      | —                                          | Получить по ID       |
| DELETE | `/users/{id}` | —      | —                                          | Удалить пользователя |

---

## Бизнес-правила

1. **Email уникален** (case-sensitive, constraint на БД). Повторный → 400.
2. **Имя не пустое** (@NotBlank валидация).
3. **При удалении пользователя** данные остаются:
   - `image_types.created_by_user_id` → NULL
   - `styles.created_by_user_id` → NULL
   - `generation_batches.owner_user_id` → NULL
   - Избранное пользователя удаляется (CASCADE)
   - Assets остаются (нет прямой FK от assets к users в новой схеме)

## Связи

```
users
  ├─→ image_types.created_by_user_id   (ON DELETE SET NULL)
  ├─→ styles.created_by_user_id        (ON DELETE SET NULL)
  ├─→ generation_batches.owner_user_id (ON DELETE SET NULL)
  ├─→ user_favorite_types.user_id      (ON DELETE CASCADE)
  └─→ user_favorite_styles.user_id     (ON DELETE CASCADE)
```
