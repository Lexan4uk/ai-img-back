# 06. Ассеты (assets)

## Назначение

Ассет — результат успешной генерации: ссылка на файл изображения + привязка
к типу и стилю. Это основная единица контента системы.

Ассеты от **всех пользователей** отображаются в общей галерее.

---

## Структура таблицы

```sql
CREATE TABLE assets (
    id            BIGSERIAL   PRIMARY KEY,
    image_type_id BIGINT      NOT NULL REFERENCES image_types(id),
    style_id      BIGINT      NOT NULL REFERENCES styles(id),
    file_uri      TEXT        NOT NULL,
    created_at    TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX ix_assets_type       ON assets(image_type_id);
CREATE INDEX ix_assets_style      ON assets(style_id);
CREATE INDEX ix_assets_type_style ON assets(image_type_id, style_id, id DESC);
```

| Поле            | Тип         | Обязательно | Описание                         |
| --------------- | ----------- | ----------- | -------------------------------- |
| `id`            | BIGSERIAL   | auto        | Уникальный идентификатор         |
| `image_type_id` | BIGINT      | ✅          | Тип изображения                  |
| `style_id`      | BIGINT      | ✅          | Стиль                            |
| `file_uri`      | TEXT        | ✅          | Путь к файлу изображения         |
| `created_at`    | TIMESTAMPTZ | auto        | Время создания                   |

---

## Связь с генерацией

Ассет — лёгкая запись. Детали генерации получаются через JOIN:

| Данные                       | Источник                                                                          |
| ---------------------------- | --------------------------------------------------------------------------------- |
| Владелец                     | `generation_batches.owner_user_id` (через `generation_requests.batch_id`)         |
| Пользовательский промпт      | `generation_batches.user_prompt`                                                  |
| Финальный промпт             | Пересобирается из `batch.user_prompt` + `type.type_prompt` + `style.style_prompt` |
| Хеш финального промпта       | `generation_requests.final_prompt_hash`                                           |
| Провайдер, модель            | `generation_batches.provider`, `generation_batches.model`                         |

## API

```
GET /assets?imageTypeId={id}&page=0&size=20                 — первая страница ассетов типа
GET /assets?imageTypeId={id}&styleId={id}&page=0&size=20    — с фильтром по стилю
```

Параметры пагинации: `page` (номер страницы, с 0) и `size` (записей на страницу, по умолчанию 20).

Header `UserId` **не нужен** — галерея общедоступна.

Сортировка: `ORDER BY id DESC` (новые первые, BIGSERIAL = хронологический порядок).

## Бизнес-правила

1. **Ассеты создаются только через генерацию** (`POST /generations`). Нет отдельного API для создания.
2. **Галерея — cross-user**: отображаются ассеты всех пользователей.
3. **При удалении типа/стиля**: `image_type_id` / `style_id` переназначается на «Неопределённый» (id=1). Ассет не удаляется.
4. **Сортировка по `id DESC`** вместо `created_at DESC` — BIGSERIAL гарантирует хронологический порядок, индекс на `(image_type_id, style_id, id DESC)` оптимизирует запрос галереи.
