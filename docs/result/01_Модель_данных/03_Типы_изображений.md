# 03. Типы изображений (image_types)

## Назначение

Общий справочник типов изображений (иконка, фотография, картина и т.д.).
Каждый тип содержит `type_prompt` — фрагмент текста, который подставляется
в начало финального промпта при генерации.

Типы видны **всем пользователям**. Создать может любой, удалить — только создатель.

---

## Структура таблицы

```sql
CREATE TABLE image_types (
    id                 BIGSERIAL PRIMARY KEY,
    created_by_user_id BIGINT    REFERENCES users(id) ON DELETE SET NULL,
    name               TEXT      NOT NULL,
    type_prompt        TEXT
);
CREATE UNIQUE INDEX uq_image_types_name ON image_types(LOWER(name));
CREATE INDEX ix_image_types_created_by ON image_types(created_by_user_id);
```

| Поле                 | Тип       | Обязательно | Описание                                                 |
| -------------------- | --------- | ----------- | -------------------------------------------------------- |
| `id`                 | BIGSERIAL | auto        | Уникальный идентификатор                                 |
| `created_by_user_id` | BIGINT    | —           | Кто создал (NULL = системный или удалённый пользователь) |
| `name`               | TEXT      | ✅          | Название типа (уникальное, case-insensitive)             |
| `type_prompt`        | TEXT      | —           | Текст для AI (например "a photograph of")                |

---

## Seed-данные

```sql
INSERT INTO image_types (name, type_prompt) VALUES ('Неопределённый', NULL);
-- id = 1 → UNDEFINED_TYPE_ID
```

## API

| Метод  | URL                          | UserId | Body                                             | Описание                    |
| ------ | ---------------------------- | ------ | ------------------------------------------------ | --------------------------- |
| POST   | `/image-types`               | ✅     | `{ "name": "Фото", "typePrompt": "a photo of" }` | Создать тип                 |
| GET    | `/image-types`               | —      | —                                                | Получить все типы           |
| DELETE | `/image-types/{id}`          | ✅     | —                                                | Удалить тип                 |
| POST   | `/image-types/{id}/favorite` | ✅     | —                                                | Добавить в избранное        |
| DELETE | `/image-types/{id}/favorite` | ✅     | —                                                | Убрать из избранного        |
| GET    | `/image-types/favorites`     | ✅     | —                                                | Получить ID избранных типов |

## Бизнес-правила

1. **Имя уникально** (case-insensitive через `LOWER(name)` index). Дубликат → 400.
2. **Имя не пустое** — сервис тримит и проверяет blank → 400.
3. **«Неопределённый» (id=1)** — неудаляемый. Попытка удалить → 403.
4. **Удалить может только создатель** (`created_by_user_id` = текущий UserId). Чужой → 403.
5. **При удалении типа** (в одной транзакции):
   - `assets.image_type_id` → переназначается на `UNDEFINED_TYPE_ID` (1)
   - `generation_requests.image_type_id` → переназначается на 1
   - `user_favorite_types` — удаляется автоматически (CASCADE)
   - Затем `DELETE FROM image_types WHERE id = :id`

## Избранное

Реализовано через таблицу `user_favorite_types` (см. [05_Избранное.md](05_Избранное.md)).

- `POST .../favorite` — `INSERT ... ON CONFLICT DO NOTHING` (идемпотентно)
- `DELETE .../favorite` — удаление записи
- `GET .../favorites` — возвращает `List<Long>` (массив ID типов)
