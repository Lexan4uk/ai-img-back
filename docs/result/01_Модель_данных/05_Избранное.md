# 05. Избранное (favorites)

## Назначение

Пользователь может отмечать типы изображений и стили как «избранные».
Избранное используется на клиенте для быстрого доступа к часто используемым
элементам (на экранах галереи и генерации).

Реализовано через две таблицы-связки (many-to-many).

---

## Структура таблиц

### user_favorite_types

```sql
CREATE TABLE user_favorite_types (
    user_id       BIGINT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    image_type_id BIGINT NOT NULL REFERENCES image_types(id) ON DELETE CASCADE,
    PRIMARY KEY (user_id, image_type_id)
);
CREATE INDEX ix_fav_types_type ON user_favorite_types(image_type_id);
```

### user_favorite_styles

```sql
CREATE TABLE user_favorite_styles (
    user_id  BIGINT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    style_id BIGINT NOT NULL REFERENCES styles(id) ON DELETE CASCADE,
    PRIMARY KEY (user_id, style_id)
);
CREATE INDEX ix_fav_styles_style ON user_favorite_styles(style_id);
```

---

## Поведение CASCADE

| Событие             | Эффект                                                |
| ------------------- | ----------------------------------------------------- |
| Удалён пользователь | Все его записи избранного удаляются                   |
| Удалён тип          | Записи `user_favorite_types` с этим типом удаляются   |
| Удалён стиль        | Записи `user_favorite_styles` с этим стилем удаляются |

---

## API

### Типы

| Метод  | URL                          | Описание                                   |
| ------ | ---------------------------- | ------------------------------------------ |
| POST   | `/image-types/{id}/favorite` | Добавить тип в избранное (идемпотентно)    |
| DELETE | `/image-types/{id}/favorite` | Убрать тип из избранного                   |
| GET    | `/image-types/favorites`     | Получить `List<Long>` — ID избранных типов |

### Стили

| Метод  | URL                     | Описание                                    |
| ------ | ----------------------- | ------------------------------------------- |
| POST   | `/styles/{id}/favorite` | Добавить стиль в избранное (идемпотентно)   |
| DELETE | `/styles/{id}/favorite` | Убрать стиль из избранного                  |
| GET    | `/styles/favorites`     | Получить `List<Long>` — ID избранных стилей |

Все запросы требуют header `UserId`.

---

## Бизнес-правила

1. **Добавление идемпотентно**: `INSERT ... ON CONFLICT DO NOTHING`. Повторный вызов — 200, без ошибки.
2. **Удаление идемпотентно**: если записи нет — 200, без ошибки.
3. **Проверка существования**: при добавлении проверяется, что тип/стиль существует → 404 если нет.
4. **Формат ответа**: `GET .../favorites` возвращает плоский массив `[1, 5, 12]` (ID, не объекты).
