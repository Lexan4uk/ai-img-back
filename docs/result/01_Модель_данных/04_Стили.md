# 04. Стили (styles)

## Назначение

Общий справочник стилей (Айвазовский, помпейская фреска, Elden Ring, Warcraft и т.д.).
Каждый стиль содержит `style_prompt` — фрагмент текста для промпта AI.

Стили — зеркальная копия типов изображений по структуре и бизнес-правилам.

---

## Структура таблицы

```sql
CREATE TABLE styles (
    id                 BIGSERIAL PRIMARY KEY,
    created_by_user_id BIGINT    REFERENCES users(id) ON DELETE SET NULL,
    name               TEXT      NOT NULL,
    style_prompt       TEXT
);
CREATE UNIQUE INDEX uq_styles_name ON styles(LOWER(name));
CREATE INDEX ix_styles_created_by ON styles(created_by_user_id);
```

| Поле                 | Тип       | Обязательно | Описание                                             |
| -------------------- | --------- | ----------- | ---------------------------------------------------- |
| `id`                 | BIGSERIAL | auto        | Уникальный идентификатор                             |
| `created_by_user_id` | BIGINT    | —           | Кто создал                                           |
| `name`               | TEXT      | ✅          | Название стиля (уникальное, case-insensitive)        |
| `style_prompt`       | TEXT      | —           | Текст для AI (например "in the style of Aivazovsky") |

---

## Seed-данные

```sql
INSERT INTO styles (name, style_prompt) VALUES ('Неопределённый', NULL);
-- id = 1 → UNDEFINED_STYLE_ID
```

## API

| Метод  | URL                     | UserId | Body                                                              | Описание                     |
| ------ | ----------------------- | ------ | ----------------------------------------------------------------- | ---------------------------- |
| POST   | `/styles`               | ✅     | `{ "name": "Айвазовский", "stylePrompt": "style of Aivazovsky" }` | Создать стиль                |
| GET    | `/styles`               | —      | —                                                                 | Получить все стили           |
| DELETE | `/styles/{id}`          | ✅     | —                                                                 | Удалить стиль                |

## Бизнес-правила

Идентичны типам изображений (см. [03*Типы*изображений.md](03_Типы_изображений.md)):

1. Имя уникально (case-insensitive). Дубликат → 400.
2. Имя не пустое.
3. «Неопределённый» (id=1) — неудаляемый → 403.
4. Удалить может только создатель → 403 для чужих.
5. При удалении: переназначение assets/requests на id=1, CASCADE избранного.

## Потенциальное расширение: группировка

Из `mod1-raw.md`: стили можно привязывать к **эпохам**, **странам**
или **произвольным направлениям** (для современных стилей вроде Elden Ring).

> В текущей версии **не реализовано**. Предполагает добавление таблиц
> `epochs`, `countries` и/или `style_tags` с many-to-many связью.
