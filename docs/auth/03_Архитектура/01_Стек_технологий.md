## Серверный стек (auth-сервер)

| Технология            | Версия | Назначение                                            |
| --------------------- | ------ | ----------------------------------------------------- |
| Java                  | 21     | Язык, LTS                                             |
| Spring Boot           | 3.5    | Веб-фреймворк, DI, автоконфигурация, @Scheduled       |
| Spring Data JDBC      | —      | Data-access (NamedParameterJdbcTemplate)              |
| spring-security-crypto | —     | BCryptPasswordEncoder (хеширование паролей)           |
| PostgreSQL            | 15+    | СУБД (общая с основным бэкендом)                      |
| Liquibase             | —      | Миграции БД (auth_users, token_blacklist)             |
| JWT-библиотека        | —      | Генерация и валидация JWT (подпись, claims, exp)      |
| Jackson               | —      | JSON-сериализация                                     |
| Lombok                | —      | Boilerplate reduction                                 |
| Maven                 | —      | Сборка                                                |

### Обоснование выбора

- **spring-security-crypto** — отдельный модуль (~100 КБ), предоставляет `BCryptPasswordEncoder`. Не тянет Spring Security целиком.
- **JWT-библиотека** — генерация access/refresh токенов, валидация подписи, проверка exp и jti. Конкретная библиотека (jjwt, nimbus-jose-jwt, auth0-java-jwt) определяется при реализации.
- **@Scheduled** — очистка `token_blacklist` (раз в день). Встроен в Spring Boot, не требует дополнительных зависимостей.

## Стек основного бэкенда (auth-компоненты)

Компоненты, добавляемые в существующий основной бэкенд для интеграции с auth-сервером.

| Технология | Версия | Назначение                                               |
| ---------- | ------ | -------------------------------------------------------- |
| Caffeine   | 3.x    | In-memory кэш валидации токенов (TTL 10 мин)            |
| OkHttp     | 4.12   | HTTP-клиент для запросов к auth-серверу                  |
| Jackson    | —      | Парсинг ответов auth-сервера                             |

### Обоснование выбора

- **Caffeine** — высокопроизводительный in-memory кэш. Используется для кэширования ответов `/validate` (ключ = SHA-256 токена, TTL = 10 мин). Снижает HTTP-нагрузку на auth-сервер.
- **OkHttp** — уже используется в проекте (client-utils). Применяется для HTTP-вызовов к auth-серверу из `JwtAuthFilter` и `AuthController`.

## Клиентский стек (auth-компоненты)

Компоненты, добавляемые в существующий десктоп-клиент.

| Технология       | Назначение                                                   |
| ---------------- | ------------------------------------------------------------ |
| auth-client-utils | Новый модуль: OkHttp Interceptor, хранение токенов, auto-refresh |
| OkHttp Interceptor | Подстановка `Authorization: Bearer` + auto-refresh при 401 |
| Preferences      | Хранение refreshToken между запусками                        |
| Base64           | Декодирование payload accessToken (получение claims)         |

## Конфигурация auth-сервера

```properties
# --- Сервер ---
server.port=9090

# --- БД (общая с основным бэкендом) ---
spring.datasource.url=jdbc:postgresql://localhost:5432/ai-img-back
spring.datasource.username=postgres
spring.datasource.password=postgres
spring.liquibase.change-log=classpath:db/changelog-master.xml

# --- JWT ---
jwt.signing-key=${JWT_SIGNING_KEY}
jwt.access-token-ttl-minutes=60
jwt.refresh-token-ttl-days=30
```

### Секреты

| Переменная       | Назначение           | Хранение              |
| ---------------- | -------------------- | --------------------- |
| `JWT_SIGNING_KEY` | Ключ подписи JWT    | переменная окружения  |

Ключ подписи хранится **только на auth-сервере**. Основной бэкенд не знает ключ — валидирует токены через HTTP-запрос `/validate`.
