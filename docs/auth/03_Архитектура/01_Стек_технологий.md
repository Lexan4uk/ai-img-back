## Стек основного бэкенда (auth-компоненты)

Компоненты, добавляемые в существующий основной бэкенд для интеграции с auth-сервером.

| Технология               | Версия | Назначение                                                |
| ------------------------ | ------ | --------------------------------------------------------- |
| Spring Security          | 6.x    | SecurityFilterChain, OncePerRequestFilter, SecurityContext |
| JJWT                     | 0.12.6 | Локальная валидация JWT (подпись, exp)                     |
| OkHttp                   | 4.12   | HTTP-клиент для проксирования auth-запросов                |

### Обоснование выбора

- **Spring Security** — индустриальный стандарт авторизации в Spring-экосистеме. `SecurityFilterChain` декларативно задаёт правила доступа. `OncePerRequestFilter` обеспечивает единообразный перехват запросов. `SecurityContext` даёт доступ к данным пользователя из любого слоя без передачи через параметры. Поддерживает conditional-конфигурацию (`auth.enabled=true/false`) для переключения между режимами разработки и продакшена.
- **JJWT** — тот же signing key, что и у auth-сервера. Локальная валидация JWT мгновенна — HTTP-вызов к auth-серверу для каждого запроса не нужен.
- **OkHttp** — уже используется в проекте. Применяется для проксирования auth-запросов (login, register, refresh, logout, change-password) к auth-серверу.

## Конфигурация auth-сервера

```properties
# --- Сервер ---
server.port=9090

# --- БД (общая с основным бэкендом) ---
spring.datasource.url=jdbc:postgresql://localhost:5432/ai-img-back
spring.datasource.username=postgres
spring.datasource.password=postgres
spring.liquibase.change-log=classpath:db/auth-changelog-master.xml

# --- JWT ---
jwt.signing-key=${JWT_SIGNING_KEY:default-dev-secret-key-change-in-production-must-be-at-least-32-bytes}
jwt.access-token-ttl-minutes=60
jwt.refresh-token-ttl-days=30
```

## Конфигурация основного бэкенда (auth-компоненты)

```properties
# --- Auth ---
auth.server.url=http://localhost:9090
auth.enabled=true
jwt.signing-key=${JWT_SIGNING_KEY:default-dev-secret-key-change-in-production-must-be-at-least-32-bytes}
```

### Секреты

| Переменная        | Назначение           | Где используется                | Хранение              |
| ----------------- | -------------------- | ------------------------------- | --------------------- |
| `JWT_SIGNING_KEY` | Ключ подписи JWT     | auth-сервер + основной бэкенд   | переменная окружения  |

Ключ подписи **общий** для auth-сервера и основного бэкенда. Это позволяет основному бэкенду валидировать JWT локально (без HTTP-вызова к auth-серверу на каждый запрос).

### Режимы работы

| Режим      | `auth.enabled` | Поведение                                                |
| ---------- | -------------- | -------------------------------------------------------- |
| Production | `true`         | Spring Security активен, JWT-фильтр проверяет все запросы |
| Development | `false`       | Spring Security отключён, все запросы разрешены            |
