# S-AUTH-01. Логин

## Назначение

Экран авторизации. Пользователь вводит email и пароль, получает JWT-токены и переходит на главный экран приложения. Экран показывается при запуске (если нет сохранённого refreshToken или refresh не удался) и после logout.

## Элементы

| Элемент                 | Описание                                                    |
| ----------------------- | ----------------------------------------------------------- |
| Заголовок               | «AI Image Library» — название приложения                    |
| Поле email              | Текстовое поле, placeholder «user@example.com»              |
| Поле пароль             | Текстовое поле с маской, кнопка [👁] для показа/скрытия     |
| Ошибка                  | Текст ошибки под полями (скрыт, пока нет ошибки)            |
| Кнопка «Войти»          | Primary-кнопка на всю ширину формы                          |
| Ссылка «Зарегистрироваться» | Навигация на экран регистрации (S-AUTH-02)              |

---

## Поля формы

### `email`

| Факт             | Детали                                                        |
| ---------------- | ------------------------------------------------------------- |
| Лимит            | 254 символа (поле не принимает больше)                        |
| Валидация        | не пустое + формат email (regex `^[^\s@]+@[^\s@]+\.[^\s@]+$`) |
| Trim             | клиент тримит перед отправкой                                 |
| Ошибка валидации | подсветка поля красным (без текста — только при отправке)     |

### `password`

| Факт             | Детали                                       |
| ---------------- | -------------------------------------------- |
| Лимит            | 128 символов (поле не принимает больше)       |
| Валидация        | не пустое, минимум 6 символов                |
| Trim             | нет — пароль передаётся как есть             |
| Показ/скрытие    | кнопка [👁] переключает маску               |
| Ошибка валидации | подсветка поля красным                       |

## Состояния

### Начальное

- Поля пустые
- Кнопка «Войти» **неактивна** (disabled)
- Ошибка скрыта
- Фокус на поле email

### Валидация пройдена

Кнопка «Войти» **активна**, когда:
- email не пустой и соответствует формату
- password не пустой и ≥ 6 символов

### Загрузка

- Кнопка «Войти» показывает спиннер, disabled
- Поля disabled (нельзя редактировать)
- Ссылка «Зарегистрироваться» disabled

### Успех

- `POST /auth/login` → `{ accessToken, refreshToken }`
- Токены сохраняются (access → память, refresh → Preferences)
- Клиент декодирует accessToken → UserClaims
- Навигация на главный экран приложения

### Ошибка

| HTTP | Сообщение сервера                    | Отображение                                 |
| ---- | ------------------------------------ | ------------------------------------------- |
| 401  | «Неверный email или пароль»          | текст ошибки под полями формы               |
| 503  | «Сервер авторизации недоступен»      | toast / snackbar                            |
| сеть | timeout / connection refused         | toast «Нет подключения к серверу»           |

При ошибке 401 — поля остаются заполненными, пользователь может исправить и повторить.

## Навигация

```
Запуск приложения
    │
    ├── hasRefreshToken?
    │   ├── Да → POST /auth/refresh
    │   │        ├── 200 → Главный экран
    │   │        └── 401/503 → Логин (S-AUTH-01)
    │   └── Нет → Логин (S-AUTH-01)
    │
Логин (S-AUTH-01)
    ├── «Зарегистрироваться» → Регистрация (S-AUTH-02)
    └── Успешный логин → Главный экран
```

| Действие | Запрос                                              |
| -------- | --------------------------------------------------- |
| Логин    | `POST /auth/login` с `{ email, password }`          |
| Ответ    | `{ accessToken, refreshToken }`                     |
