# 03. Навигация и auth-элементы

## Изменения в навигации

Auth-модуль добавляет два экрана (Логин, Регистрация) и меняет общую карту навигации приложения.

### Карта навигации (с auth)

```
         ┌──────────────┐
         │  Запуск       │
         │  приложения   │
         └──────┬───────┘
                │
                ▼
         ┌──────────────┐
         │ Refresh?      │
         │ (hasRefreshToken)
         └──────┬───────┘
                │
          ┌─────┴─────┐
          │           │
       Успех       Неудача / нет токена
          │           │
          ▼           ▼
   Главный экран   ┌──────────────┐
                   │  S-AUTH-01   │
              ┌───►│  Логин       │◄───────────────────┐
              │    └──────┬───────┘                     │
              │           │                             │
              │           │ «Зарегистрироваться»        │
              │           ▼                             │
              │    ┌──────────────┐                     │
              │    │  S-AUTH-02   │                     │
              └────│  Регистрация │                     │
               «Войти»                                  │
              │    └──────┬───────┘                     │
              │           │                             │
              │       Успешный                          │
              │     логин / рег.                        │
              │           │                             │
              │           ▼                             │
              │    ┌──────────────┐                     │
              │    │  Главный     │     Logout /        │
              │    │  экран       │───  refresh failed ─┘
              │    └──────────────┘
              │
              └─── 401 + refresh failed (из любого экрана)
```

### До auth-модуля

```
Запуск → Главный экран (Галерея)
```

### После auth-модуля

```
Запуск → Refresh → Успех → Главный экран
                 → Неудача → Логин → [Регистрация] → Главный экран
```

## Диалог смены пароля

Открывается из настроек по кнопке «Сменить пароль».

### Макет

```
┌─── Смена пароля ─────────────────────────────────┐
│                                                    │
│  Текущий пароль:                                   │
│  ┌──────────────────────────────────────┐          │
│  │ ••••••••                         [👁] │          │
│  └──────────────────────────────────────┘          │
│  ⚠ Неверный текущий пароль                         │
│                                                    │
│  Новый пароль:                                     │
│  ┌──────────────────────────────────────┐          │
│  │ ••••••••                         [👁] │          │
│  └──────────────────────────────────────┘          │
│                                                    │
│  Подтверждение нового пароля:                      │
│  ┌──────────────────────────────────────┐          │
│  │ ••••••••                         [👁] │          │
│  └──────────────────────────────────────┘          │
│  ⚠ Пароли не совпадают                             │
│                                                    │
│  [ Отмена ]           [ Сменить пароль ]           │
└────────────────────────────────────────────────────┘
```

### Поля

#### `currentPassword`

| Факт             | Детали                                       |
| ---------------- | -------------------------------------------- |
| Лимит            | 128 символов (поле не принимает больше)       |
| Валидация        | не пустое, минимум 6 символов                |
| Trim             | нет — пароль передаётся как есть             |
| Показ/скрытие    | кнопка [👁]                                  |
| 401 от сервера   | «Неверный текущий пароль» — под полем        |

#### `newPassword`

| Факт             | Детали                                       |
| ---------------- | -------------------------------------------- |
| Лимит            | 128 символов                                  |
| Валидация        | не пустое, ≥ 6 символов                      |
| Trim             | нет                                           |
| Показ/скрытие    | кнопка [👁]                                  |

#### `confirmNewPassword`

| Факт             | Детали                                           |
| ---------------- | ------------------------------------------------ |
| Лимит            | 128 символов                                     |
| Валидация        | не пустое + совпадает с `newPassword`            |
| Trim             | нет                                               |
| На сервер        | **не отправляется** — проверка только на клиенте |
| Ошибка           | «Пароли не совпадают» — под полем               |

### Состояние кнопки «Сменить пароль»

Кнопка **неактивна**, пока:
- currentPassword пустой или < 6 символов
- newPassword пустой или < 6 символов
- confirmNewPassword пустой или не совпадает с newPassword

### Состояния

| Состояние | Поведение                                                      |
| --------- | -------------------------------------------------------------- |
| Загрузка  | кнопка показывает спиннер, поля disabled                       |
| Успех     | 204 → диалог закрывается, toast «Пароль изменён»              |
| 401       | «Неверный текущий пароль» — под полем `currentPassword`        |
| 503       | toast «Сервис временно недоступен»                             |
| Сеть      | toast «Нет подключения к серверу»                              |

Пояснение: если пользователь забыл текущий пароль — он обращается к админу за временным (см. бэкенд §8.5), входит с ним и меняет на свой.

### API

| Действие       | Запрос                                                            |
| -------------- | ----------------------------------------------------------------- |
| Смена пароля   | `POST /auth/change-password` с `{ currentPassword, newPassword }` |
| Ответ          | `204 No Content`                                                  |

## Кнопка Logout

### Расположение

Header — кнопка «Выйти» справа.

### Поведение

1. Клик → диалог подтверждения:

```
┌─── Выход ──────────────────────────┐
│                                      │
│  Вы уверены, что хотите выйти?      │
│                                      │
│  [ Отмена ]         [ Выйти ]       │
└──────────────────────────────────────┘
```

2. Подтверждение → `POST /auth/logout` с accessToken (заголовок) + `{ refreshToken }` (тело)
3. Независимо от результата (200 / ошибка):
   - accessToken удаляется из памяти
   - refreshToken удаляется из Preferences
   - Навигация на экран Логина (S-AUTH-01)

Logout — best-effort: даже при ошибке сети клиент очищает токены локально.

### API

| Действие | Запрос                                                                   |
| -------- | ------------------------------------------------------------------------ |
| Logout   | `POST /auth/logout` с Bearer-заголовком + `{ refreshToken }` в теле     |
| Ответ    | `204 No Content`                                                         |

## Экран загрузки (splash)

При запуске, пока выполняется `POST /auth/refresh`, показывается экран загрузки:

```
┌──────────────────────────────────────────────────────────────────────┐
│                                                                      │
│                                                                      │
│                                                                      │
│                         AI Image Library                             │
│                                                                      │
│                             ⏳                                       │
│                                                                      │
│                                                                      │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘
```

| Результат refresh | Переход                   |
| ----------------- | ------------------------- |
| 200               | Главный экран             |
| 401 / 503 / сеть  | Экран Логина (S-AUTH-01)  |
| Нет refreshToken  | Экран Логина (сразу)      |
