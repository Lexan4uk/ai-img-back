# S-AUTH-02. Регистрация

## Назначение

Экран создания нового аккаунта. Пользователь вводит email, пароль (с подтверждением) и отображаемое имя. При успехе — получает JWT-токены и переходит на главный экран. Переход с экрана логина по ссылке «Зарегистрироваться».

## Элементы

| Элемент                      | Описание                                           |
| ---------------------------- | -------------------------------------------------- |
| Заголовок                    | «AI Image Library» — название приложения           |
| Поле email                   | Текстовое поле                                     |
| Поле «Имя»                  | Текстовое поле (displayName)                       |
| Поле пароль                  | Текстовое поле с маской + [👁]                     |
| Поле подтверждение пароля    | Текстовое поле с маской + [👁]                     |
| Ошибка под полем             | Контекстная ошибка (email занят / пароли не совпадают) |
| Кнопка «Зарегистрироваться» | Primary-кнопка на всю ширину формы                 |
| Ссылка «Войти»              | Навигация на экран логина (S-AUTH-01)              |

## Поля формы

### `email`

| Факт             | Детали                                                        |
| ---------------- | ------------------------------------------------------------- |
| Лимит            | 254 символа (поле не принимает больше)                        |
| Валидация        | не пустое + формат email (regex `^[^\s@]+@[^\s@]+\.[^\s@]+$`) |
| Trim             | клиент тримит перед отправкой                                 |
| 409 от сервера   | «Email уже зарегистрирован» — текст ошибки под полем          |

### `displayName`

| Факт             | Детали                                       |
| ---------------- | -------------------------------------------- |
| Лимит            | 100 символов (поле не принимает больше)       |
| Валидация        | не пустое (после trim)                       |
| Trim             | клиент тримит перед отправкой                |

### `password`

| Факт             | Детали                                       |
| ---------------- | -------------------------------------------- |
| Лимит            | 128 символов (поле не принимает больше)       |
| Валидация        | не пустое, минимум 6 символов                |
| Trim             | нет — пароль передаётся как есть             |
| Показ/скрытие    | кнопка [👁] переключает маску               |

### `confirmPassword`

| Факт             | Детали                                           |
| ---------------- | ------------------------------------------------ |
| Лимит            | 128 символов                                     |
| Валидация        | не пустое + совпадает с `password`               |
| Trim             | нет — сравнивается как есть                      |
| На сервер        | **не отправляется** — проверка только на клиенте |
| Ошибка           | «Пароли не совпадают» — показывается под полем, когда оба поля заполнены и не совпадают |

## Состояния

### Начальное

- Поля пустые
- Кнопка «Зарегистрироваться» **неактивна** (disabled)
- Ошибки скрыты
- Фокус на поле email

### Валидация пройдена

Кнопка «Зарегистрироваться» **активна**, когда:
- email не пустой и соответствует формату
- displayName не пустой (после trim)
- password не пустой и ≥ 6 символов
- confirmPassword не пустой и совпадает с password

### Загрузка

- Кнопка «Зарегистрироваться» показывает спиннер, disabled
- Все поля disabled
- Ссылка «Войти» disabled

### Успех

- `POST /auth/register` → `{ accessToken, refreshToken }`
- Токены сохраняются (access → память, refresh → Preferences)
- Клиент декодирует accessToken → UserClaims
- Навигация на главный экран приложения

### Ошибки

| HTTP | Сообщение сервера                    | Отображение                                   |
| ---- | ------------------------------------ | --------------------------------------------- |
| 409  | «Email уже зарегистрирован»          | текст ошибки под полем email                  |
| 400  | «Невалидные данные»                  | toast / snackbar с текстом                    |
| 503  | «Сервер авторизации недоступен»      | toast / snackbar                              |
| сеть | timeout / connection refused         | toast «Нет подключения к серверу»             |

При ошибке 409 — поля остаются заполненными, пользователь может изменить email и повторить.

## API

| Действие     | Запрос                                                                 |
| ------------ | ---------------------------------------------------------------------- |
| Регистрация  | `POST /auth/register` с `{ email, password, displayName }`            |
| Ответ        | `{ accessToken, refreshToken }`                                        |

`confirmPassword` в запрос **не включается** — это клиентская проверка.
