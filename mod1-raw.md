# Библиотека стилей AI

## Общее описание

Серверное приложение с базой данных стилей искусственного интеллекта. Система предназначена для создания и хранения большой базы образцов изображений, сгенерированных AI в различных стилях. Позволяет дизайнеру, постоянно работающему с ИИ, понимать, как выглядят те или иные виды изображений в том или ином стиле.

Целевая аудитория это веб разработчики студии для скорости чтобы получали наполнения сайта.

---

## Архитектура

### Серверная часть

- Серверное приложение с базой данных **PostgreSQL**
- Подключение к **API ChatGPT** для генерации изображений
- Изображения сохраняются на диск, в БД записываются ссылки на них

### Клиентская часть (UI)

- **Локальный интерфейс** — на Kotlin Multiplatform
- **Удалённый клиентский интерфейс** — тоже на Kotlin Multiplatform
- Платформы: **Windows**, **Ubuntu**, **Android**
- Функционал локального и удалённого интерфейсов **идентичный**

---

## Сущности системы

### Типы изображений

- Примеры: иконка, фотография, картина (и, возможно, другие)
- Некоторые типы могут быть созданы по умолчанию
- Пользователь может **создавать**, **редактировать** и **удалять** типы

### Стили

- Примеры: Айвазовский, помпейская фреска, Elden Ring, Warcraft и т.д.
- Пользователь может **создавать**, **редактировать** и **удалять** стили
- Количество стилей не ограничено

### Группировка стилей

- Стили можно привязывать к **эпохам** и **странам**
- Либо привязывать к **произвольным направлениям** (например, для современных стилей вроде Elden Ring или Warcraft, которые не привязаны к эпохам)

## Процесс генерации

### Ввод данных пользователем

1. Пользователь пишет **промпт** (например: «картина, на которой изображён человек с молотом, выковывающий лопату в кузнице в горах»)
2. Выбирает один или несколько **типов изображений** (картина, иконка и т.д.)
3. Выбирает один или несколько **стилей** (возможно, несколько десятков)
4. Система формирует все комбинации (например: 5 типов × 10 стилей = 50 комбинаций)

### Отправка запросов

- Сервер **последовательно** отправляет промпты в ChatGPT API для генерации
- **Первая версия** — однопоточная (отправил → получил результат → отправил следующий)
- Архитектура должна быть **модульной**, чтобы в дальнейшем можно было:
  - Переключаться между однопоточным и многопоточным режимом
  - Регулировать количество одновременных соединений

### Обработка дубликатов

- Если комбинация **промпт + тип + стиль** уже существует в базе, система спрашивает пользователя:
  - Перезаписать существующее изображение
  - Пропустить (не генерировать заново)

## Галерея / Просмотр

### Навигация

- Отдельные экраны по типам изображений (например, экран «Иконки», экран «Фотографии» и т.д.)
- На экране типа отображаются изображения **во всех стилях** (компактная галерея)
- Каждый стиль **подписан**

### Превью и детальный просмотр

- В рамках одного стиля и одного типа может быть несколько промптов (несколько изображений)
- В галерее в качестве превью показывается **только последнее сгенерированное изображение**
- При нажатии на превью — переход в подраздел, где можно посмотреть **все примеры** для данной комбинации стиль + тип

### Доступ

- Галерея доступна как в локальном, так и в удалённом интерфейсе
